# Nova Theme Studio — 产品设计方案

> 版本 1.0 · 待确认后实施

---

## 一、入口

```
App 启动 → 如果检测到 .env 存在（dev 模式）
  → header 右侧新增 🎨 图标按钮
  → 点击进入 Theme Studio（独立全屏页面，占用整个窗口）
  → 按 Esc 或点返回箭头退出
```

**不是设置里的一个标签。** 是全屏沉浸式开发环境，和媒体库页面同级。

## 三、场景网格设计

每个场景卡片用**真实缩略图**渲染，不是 lucide icon：

```
┌──────────────────┐
│                  │
│                  │  ← 实际生成的 webp/jpg 缩略图
│                  │     如果素材不存在，显示灰色占位 + 🎬图标
│                  │
├──────────────────┤
│ ⚡ scene1  ✅    │  ← 状态指示灯
│ 1920×1080 · 2MB │  ← 分辨率 + 文件大小
└──────────────────┘
```

### 卡片覆盖层（hover 时显示）

```
┌──────────────────┐
│  👁 预览         │  ← 点击全屏预览素材
│  🔄 重新生成     │  ← 重新调用 AI
│  ✏️ 编辑提示词   │  ← 弹出编辑框
│  🗑️ 删除         │  ← 删除素材文件
└──────────────────┘
```

### 状态指示灯

| 图标 | 状态 | 含义 |
|---|---|---|
| 🟢 绿色圆点 | done | 素材已生成，可用 |
| 🔵 蓝色旋转 | generating | AI 正在生成中 |
| ⚪ 灰色圆点 | todo | 提示词已写好，待生成 |
| ⏭️ 灰色横杠 | skip | 跳过，不需要此场景 |

- 可以逐场景切换，模拟用户在 Home 页的实际体验

## 九、技术实现

### 路由

```
App.tsx 新增路由:
  /studio  → ThemeStudioPage（独立全屏页面，不经过 Layout）
```

### 文件清单

| 文件 | 作用 |
|---|---|
| `src/pages/ThemeStudio.tsx` | 独立全屏页面主组件 |
| `src/components/studio/ProjectList.tsx` | 左侧项目列表 |
| `src/components/studio/SceneGrid.tsx` | 中央场景网格 + 卡片 |
| `src/components/studio/SceneEditor.tsx` | 场景编辑模态框 |
| `src/components/studio/GeneratePanel.tsx` | 批量生成面板 |
| `src/components/studio/PreviewMode.tsx` | 全屏预览模式 |
| `src/components/studio/PublishDialog.tsx` | 打包发布对话框 |
| `src/components/studio/NewProjectDialog.tsx` | 新建项目对话框 |
| `src-tauri/src/commands/theme_studio.rs` | 已有，需扩展 |
| `src/stores/themeStudioStore.ts` | 前端状态管理 |

### 关键交互

- 双击场景卡片 → 弹出 SceneEditor
- 拖拽素材文件到场景卡片 → 手动替换素材
- Ctrl+S → 保存当前编辑
- Esc → 退出预览模式/关闭模态框
- 空格 → 预览模式下切换下一个场景

## 十、不做什么

| 不做 | 原因 |
|---|---|
| AI 提示词自动补全 | 超出范围，用户自己写 |
| 图像编辑（裁剪/调色/滤镜） | 不需要，AI 生成质量足够 |
| 多用户协作 | 单人项目 |
| 版本分支管理 | manifest.version 递增足够 |
| 在线素材库 | 本地文件足够 |
