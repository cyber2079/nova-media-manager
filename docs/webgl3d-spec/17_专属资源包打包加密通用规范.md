## 一、打包脚本

### 1.1 脚本入口

```
scripts/webgl3d/
├── pipeline.mjs             ← 全管线单文件（Node.js）
│                               --steps env|validate|manifest|pack|sign
└── output/                  ← 打包产物输出目录
```

### 1.2 打包命令

```bash
# 全管线一键执行
node scripts/webgl3d/pipeline.mjs \
  --source D:\nova-themes-assets\webgl3d\cyber-girl-apartment \
  --theme-id cyber-girl-apartment \
  --version 1.0.0 \
  --steps env,validate,manifest,pack

# 仅校验素材（不打包）
node scripts/webgl3d/pipeline.mjs \
  --source D:\nova-themes-assets\webgl3d\cyber-girl-apartment \
  --steps validate

# Ed25519 签名（需要私钥）
node scripts/webgl3d/pipeline.mjs \
  --source D:\nova-themes-assets\webgl3d\cyber-girl-apartment \
  --steps sign --sign-key <path-to-private-key>
```

### 1.3 打包流程

```
1. env — 离线工具环境校验（阻断级）
   ├─ 检查 basisu、gltf-transform、draco_encoder 是否在 PATH
   ├─ 通过 → 继续
   └─ 不通过 → 阻断，输出缺失工具提示
       │
2. validate — 素材合规校验
   ├─ 检查必需目录、文件命名规范
   ├─ sharp 检测贴图分辨率（如有）
   ├─ manifest.json 结构校验（如存在）
   ├─ 通过 → 继续
   └─ 不通过 → 阻断，输出报错日志
       │
3. manifest — 生成 Manifest JSON
   ├─ 扫描 source 目录
   ├─ 计算每个文件的 SHA256
   ├─ 合并 manifest.template.json（如存在）
   └─ 输出 manifest.json
       │
4. pack — 分块打包
   ├─ 读取 manifest 中的 resources 列表
   ├─ 每个资源独立压缩（或 store）
   ├─ 构建 Header(64B) + Manifest(gzip) + Resource Blocks
   └─ 写入 .nv3d 文件
       │
5. sign — 签名
   ├─ 对 NV3D 文件内容计算 Ed25519 签名（Node 22 Web Crypto）
   └─ 64B 签名写入 Footer
```

## 三、分块压缩策略

### 3.1 Block Type 选择规则

| 文件类型 | Block Type | 理由 |
|---------|-----------|------|
| `.glb` (Draco 压缩) | `store (0x02)` | Draco 已高效压缩，再 gzip 无效 |
| `.glb` (无压缩) | `gzip (0x01)` | 减少包体 |
| `.ktx2` | `store (0x02)` | KTX2 内置压缩 |
| `.png` / `.webp` | `store (0x02)` | 已压缩，gzip 无效 |
| `.jpg` | `store (0x02)` | 已压缩 |
| `.glsl` | `gzip (0x01)` | 纯文本，压缩率高 |
| `.json` | `gzip (0x01)` | 纯文本 |
| `.ogg` | `store (0x02)` | 已压缩音频 |

### 3.2 Block 大小建议

| 资源类型 | 建议 Block 粒度 | 说明 |
|---------|----------------|------|
| 模型 | 单文件 | 一个 .glb = 一个 Block |
| 贴图 | 单文件或分组 | 一个 .ktx2 = 一个 Block |
| 动画 | 单文件 | 一个 .glb = 一个 Block |
| i18n | 按语种 | zh.json / en.json 各自一个 Block（减少切换语言时的读取量） |
| Shader | 单文件 | 一个 .glsl = 一个 Block |
| 音频 | 单文件 | 一个 .ogg = 一个 Block |

## 四、完整性校验

### 4.1 打包时校验

| 校验项 | 工具/方法 | 阻断级别 |
|--------|----------|---------|
| 模型面数不超限 | .glb 文件大小启发式检测（> 50MB 告警） | 阻断 |
| 贴图分辨率不超限 | sharp | 阻断 |
| 文件名 kebab-case | 正则 `^[a-z0-9][a-z0-9_.-]*$` | 阻断 |
| Manifest 必填字段完整 | JSON Schema 校验 | 阻断 |
| 资源引用完整性 | 逐 `*Ref` 字段匹配 `resources` | 阻断 |
| 文件 Hash 计算 | SHA256 | 自动写入 |
| i18n 语种覆盖 | 检查至少 zh+en | 警告 |
| 预览图分辨率 | Pillow 检查尺寸 | 警告（不阻断） |
| Shader 编译超时检测 | 3s 阈值，超时则中止编译 | 阻断 |

### 4.2 加载时校验（客户端侧）

| 校验项 | 行为 |
|--------|------|
| NV3D Ed25519 签名 (Footer) | 无效 → 拒绝加载 |
| Manifest Hash (Header vs Footer) | 不匹配 → 拒绝加载 |
| 每个 Block 的 SHA256 | 不匹配 → 该 Block 视为损坏 |
| Manifest JSON Schema | 不匹配 → 拒绝加载 |

## 五、加密方案

### 5.1 加密算法

NV3D 使用 AES-256-GCM 加密，密钥派生复用现有体系（编译种子 + theme_id）：

- flags bit 0 标记加密状态
- 加密范围：Manifest + Resource Blocks（Header 和 Footer 保持明文）
- 密钥不进入公库（已在现有安全方案中）

### 5.2 密钥管理

- 与现有普通主题共用密钥派生体系
- 不新增密钥存储位置
- 私钥不进入公库（已在现有安全方案中）

## 六、打包产物

### 6.1 输出目录结构

```
scripts/webgl3d/output/
├── cyber-girl-apartment_v1.0.0.nv3d     ← 打包完成的 NV3D 文件
├── cyber-girl-apartment_v1.0.0.manifest.json  ← 打包时生成的完整 Manifest（调试用）
└── cyber-girl-apartment_v1.0.0.checksums.txt   ← 所有 Block 的 SHA256 列表
```

### 6.2 产物命名

```
{theme_id}_v{version}.nv3d

示例：
cyber-girl-apartment_v1.0.0.nv3d
ink-wash_v1.0.0.nv3d
```

### 6.3 产物分发

- 上传至 CDN（`https://scm-think.cn/themes/webgl3d/`）
- 服务端元数据 API 返回下载 URL + 文件大小 + SHA256
- 客户端下载后校验 SHA256 是否与 API 返回一致

## 七、素材自动化校验工具

### 7.1 pipeline.mjs --steps validate 检查项

校验逻辑位于 `scripts/webgl3d/pipeline.mjs` 的 `stepValidate()` 函数中：

| 检查项 | 方法 | 阻断级别 |
|--------|------|---------|
| 必需目录完整性 | `fs.existsSync` 检查 models/ textures/ preview/ i18n | 阻断 |
| 文件命名规范 | 正则 `/^[a-z0-9][a-z0-9_.-]*$/` | 阻断 |
| 贴图分辨率 | `sharp` metadata（可选依赖，缺失时跳过） | 超 4096 阻断 |
| .glb 文件大小 | 启发式：> 50MB 告警（可能面数超限） | 警告 |
| manifest 必填字段 | JSON.parse + 逐字段检查 | 阻断 |
| i18n 语种覆盖 | 检查 zh/en JSON 文件存在 | 警告 |
| 贴图可读性 | sharp 尝试读取，损坏则阻断 | 阻断 |

### 7.2 输出格式

```
  [validate — 素材合规校验]
  OK  目录存在: models/
  OK  目录存在: textures/
  FAIL NV3D_ASST_NAMING: models/Scene_High.glb
  FAIL NV3D_ASST_RESOLUTION: textures/scene/diffuse.png — 8192×8192 超过 4096
  WARN i18n/ko.json 缺失
  2 errors, 1 warnings
```
- 退出码非 0 → 打包脚本中断
