## 一、打包脚本

### 1.1 脚本入口

```
scripts/webgl3d/
├── env_check.py             ← 离线工具版本校验（打包前置）
├── pack_nv3d.py             ← 主打包脚本
├── validate_assets.py       ← 素材合规校验（打包前置）
├── sign_nv3d.py             ← 签名工具
├── manifest_generator.py    ← Manifest 自动生成
├── requirements.txt         ← Python 依赖
└── output/                  ← 打包产物输出目录
```

### 1.2 打包命令

```bash
# 完整打包流程
python scripts/webgl3d/pack_nv3d.py \
  --source D:\nova-themes-assets\webgl3d\cyber-girl-apartment \
  --output scripts/webgl3d/output \
  --theme-id cyber-girl-apartment \
  --version 1.0.0 \
  --sign-key <path-to-private-key>

# 仅校验素材（不打包）
python scripts/webgl3d/validate_assets.py \
  --source D:\nova-themes-assets\webgl3d\cyber-girl-apartment
```

### 1.3 打包流程

```
0. env_check.py — 离线工具环境校验（阻断级）
   ├─ 检查 Python、Pillow、basisu、gltf-transform、draco_encoder 版本
   ├─ 通过 → 继续
   └─ 不通过 → 阻断，输出缺失工具和预期版本
       │
1. validate_assets.py — 素材合规校验
   ├─ 通过 → 继续
   └─ 不通过 → 阻断，输出报错日志
       │
2. manifest_generator.py — 生成 Manifest JSON
   ├─ 扫描 source 目录
   ├─ 计算每个文件的 SHA256
   ├─ 写入资源列表 + 元数据
   └─ 输出 manifest.json
       │
3. 分块打包
   ├─ 读取 manifest 中的 resources 列表
   ├─ 每个资源独立压缩（或 store）
   ├─ 构建 Header + Manifest + Resource Blocks
   └─ 写入 .nv3d 文件
       │
4. sign_nv3d.py — 签名
   ├─ 对 NV3D 文件内容计算 Ed25519 签名
   └─ 64B 签名写入 Footer
```

## 三、分块压缩策略

### 3.1 Block Type 选择规则

| 文件类型 | Block Type | 理由 |
|---------|-----------|------|
| `.glb` (Draco 压缩) | `store (0x02)` | Draco 已高效压缩，再 gzip 无效 |
| `.glb` (无压缩) | `gzip (0x01)` | 减少包体 |
| `.ktx2` | `store (0x02)` | KTX2 内置压缩 |
| `.png` / `.webp` | `store (0x02)` | 已压缩，gzip 无效 |
| `.jpg` | `store (0x02)` | 已压缩 |
| `.glsl` | `gzip (0x01)` | 纯文本，压缩率高 |
| `.json` | `gzip (0x01)` | 纯文本 |
| `.ogg` | `store (0x02)` | 已压缩音频 |

### 3.2 Block 大小建议

| 资源类型 | 建议 Block 粒度 | 说明 |
|---------|----------------|------|
| 模型 | 单文件 | 一个 .glb = 一个 Block |
| 贴图 | 单文件或分组 | 一个 .ktx2 = 一个 Block |
| 动画 | 单文件 | 一个 .glb = 一个 Block |
| i18n | 按语种 | zh.json / en.json 各自一个 Block（减少切换语言时的读取量） |
| Shader | 单文件 | 一个 .glsl = 一个 Block |
| 音频 | 单文件 | 一个 .ogg = 一个 Block |

## 四、完整性校验

### 4.1 打包时校验

| 校验项 | 工具/方法 | 阻断级别 |
|--------|----------|---------|
| 模型面数不超限 | glTF-Validator + Python 解析 | 阻断 |
| 贴图分辨率不超限 | Pillow | 阻断 |
| 文件名 kebab-case | 正则 `^[a-z0-9][a-z0-9_.-]*$` | 阻断 |
| Manifest 必填字段完整 | JSON Schema 校验 | 阻断 |
| 资源引用完整性 | 逐 `*Ref` 字段匹配 `resources` | 阻断 |
| 文件 Hash 计算 | SHA256 | 自动写入 |
| i18n 语种覆盖 | 检查至少 zh+en | 警告 |
| 预览图分辨率 | Pillow 检查尺寸 | 警告（不阻断） |
| Shader 编译超时检测 | 3s 阈值，超时则中止编译 | 阻断 |

### 4.2 加载时校验（客户端侧）

| 校验项 | 行为 |
|--------|------|
| NV3D Ed25519 签名 (Footer) | 无效 → 拒绝加载 |
| Manifest Hash (Header vs Footer) | 不匹配 → 拒绝加载 |
| 每个 Block 的 SHA256 | 不匹配 → 该 Block 视为损坏 |
| Manifest JSON Schema | 不匹配 → 拒绝加载 |

## 五、加密方案

### 5.1 加密算法

NV3D 使用 AES-256-GCM 加密，密钥派生复用现有体系（编译种子 + theme_id）：

- flags bit 0 标记加密状态
- 加密范围：Manifest + Resource Blocks（Header 和 Footer 保持明文）
- 密钥不进入公库（已在现有安全方案中）

### 5.2 密钥管理

- 与现有普通主题共用密钥派生体系
- 不新增密钥存储位置
- 私钥不进入公库（已在现有安全方案中）

## 七、素材自动化校验工具

### 7.1 validate_assets.py 检查项

```python
# 伪代码
def validate(source_dir: str) -> list[ValidationError]:
    errors = []

    # 1. 目录结构完整性
    errors += check_required_dirs(source_dir, ["models", "textures", "preview", "i18n"])

    # 2. 文件命名规范
    for file in iter_files(source_dir):
        if not re.match(r'^[a-z0-9][a-z0-9_.-]*$', file.name):
            errors.append(ValidationError(file, "命名不规范"))

    # 3. 模型面数
    for glb in glob("models/**/*.glb"):
        face_count = get_face_count(glb)
        if face_count > 100000:  # 硬性上限
            errors.append(ValidationError(glb, f"面数超限: {face_count}"))

    # 4. 贴图分辨率
    for tex in glob("textures/**/*.{png,ktx2,webp}"):
        resolution = get_resolution(tex)
        if resolution[0] > 4096 or resolution[1] > 4096:
            errors.append(ValidationError(tex, f"分辨率超限: {resolution}"))

    # 5. Manifest 完整性
    manifest = load_manifest(source_dir)
    errors += validate_manifest_schema(manifest)

    return errors
```

### 7.2 输出格式

```
ERROR: models/Scene_High.glb — 命名不规范（含大写字母）
ERROR: textures/scene/diffuse.png — 分辨率 8192×8192 超限
WARNING: i18n/ko.json — 缺失 recommended
PASS: 142 files, 3 errors, 1 warning
```

- `ERROR` → 阻断打包
- `WARNING` → 不阻断，但输出提醒
- 退出码非 0 → 打包脚本中断
