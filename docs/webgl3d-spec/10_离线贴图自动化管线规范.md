## 二、脚本开发规则

### 2.1 技术选型

- **语言**：Python 3
- **图片处理**：Pillow（PIL）
- **KTX2 转换**：basisu CLI（开源工具，本地运行）
- **Draco 压缩**：gltf-transform CLI 或 draco_encoder CLI
- **脚本位置**：`scripts/webgl3d/`

### 2.2 脚本清单

| 脚本 | 用途 | 输入 | 输出 |
|------|------|------|------|
| `convert_textures.py` | PNG → KTX2 批量转换 | `*.png` | `*.ktx2` |
| `generate_mipmaps.py` | 自动生成 mipmap 层级 | `*.png` | `*.ktx2`（含 mip） |
| `resize_textures.py` | 按规范缩放分辨率 | `*.png` | 缩放后的 `*.png` |
| `compress_models.py` | glTF → Draco GLB | `*.gltf`/`*.glb` | `*.glb`（Draco 压缩） |
| `generate_lods.py` | 自动生成模型 LOD | `*_hd.glb` | `*_low.glb` |
| `validate_assets.py` | 全面质量校验 | 整个源目录 | 校验报告 |
| `pipeline.py` | 一键执行全管线 | 源目录 | 处理 + 校验报告 |

### 2.3 命令行接口

```bash
# 单个步骤
python scripts/webgl3d/convert_textures.py \
  --input D:\nova-themes-assets\webgl3d\cyber-girl-apartment\textures \
  --format ktx2 \
  --quality uastc

# 全管线一键执行
python scripts/webgl3d/pipeline.py \
  --source D:\nova-themes-assets\webgl3d\cyber-girl-apartment \
  --steps convert,compress,validate
```

## 三、纹理转换流程

### 3.1 PNG → KTX2

```
输入：PNG（RGBA8 或 RGB8）
    ↓
步骤 1：色彩空间转换
    - Base Color / Emissive → sRGB
    - Normal / Metallic-Roughness → Linear
    ↓
步骤 2：生成 Mipmap 层级（完整 mip 链）
    ↓
步骤 3：Basis Universal 编码
    - UASTC：高质量（≈3-5 bpp）
    - ETC1S：高压缩（≈1-1.5 bpp），Normal 贴图推荐
    ↓
输出：KTX2 文件
```

### 3.2 分辨率缩放策略

| 源分辨率 | 目标分辨率 | 行为 |
|---------|-----------|------|
| > 4096 | 4096 | 缩放到 4096，输出警告 |
| 2048-4096 | 保持 | — |
| < 2048 | 保持 | — |

- 放大操作禁止（不通过算法补像素）
- 功率-of-two（POT）非强制，但推荐

## 四、模型压缩流程

### 4.1 glTF → Draco GLB

```
输入：glTF 2.0（.gltf + .bin + 贴图）
    ↓
步骤 1：glTF-Transform 处理
    - 合并重复顶点
    - 移除未使用数据
    ↓
步骤 2：Draco 压缩
    - compressionLevel: 7（默认）
    - 保留 UV 和 Normal（不量化过激）
    ↓
步骤 3：打包 GLB
    - 嵌入压缩后几何体
    - 保留原贴图引用
    ↓
输出：Draco-compressed .glb
```

### 4.2 LOD 生成

```
输入：*_hd.glb（高精度模型）
    ↓
步骤 1：简化几何体（目标面数 = HD 的 15-20%）
    - 保留 UV 边界
    - 保留法线特征边
    ↓
步骤 2：简化材质
    - 贴图缩放至 HD 的 50% 分辨率
    - 移除 alpha 细节（如有）
    ↓
输出：*_low.glb
```

## 五、自动化校验集成

### 5.1 管线内校验点

| 步骤 | 校验内容 | 失败行为 |
|------|---------|---------|
| 格式转换后 | 输出文件存在 + 非 0 字节 | 跳过该文件，继续下一个 |
| 缩放后 | 分辨率符合目标 | 跳过，标记警告 |
| 压缩后 | 面数符合预算 | 跳过，标记警告 |
| 全管线完成后 | 运行 `validate_assets.py` 全面检查 | 输出报告 |

### 5.2 校验报告格式

```
=== WebGL3D Asset Validation Report ===
Theme: cyber-girl-apartment
Date: 2026-07-22 14:30:00

SUMMARY: 142 files checked, 3 errors, 2 warnings

ERRORS:
  [E001] models/scene_hd.glb — 三角面数 125000 超过上限 100000
  [E002] textures/character_diffuse.ktx2 — 分辨率为 0×0（损坏文件）
  [E003] models/props/item_03.glb — 缺少 UV 通道

WARNINGS:
  [W001] textures/props/prop_01.png — 非推荐格式 KTX2
  [W002] i18n/ko.json — 翻译覆盖率 87%（推荐 ≥ 90%）

Exit code: 1 (errors present)
```

### 5.3 CI 集成

校验脚本可在 CI 中运行：

```yaml
# .github/workflows/validate-theme.yml（示例）
- name: Validate Theme Assets
  run: python scripts/webgl3d/validate_assets.py --source ${{ inputs.theme_path }}
```

- 校验不通过 → CI 失败 → 不触发打包

## 六、输出目录约定

```
处理后素材输出到：
D:\nova-themes-assets\webgl3d\{theme_id}\processed\
    ├── models/        ← 压缩后的 .glb 文件
    ├── textures/      ← 转换后的 .ktx2 文件
    └── report.json    ← 处理报告（耗时、压缩率、文件变化）
```

原始源文件保留在：
```
D:\nova-themes-assets\webgl3d\{theme_id}\source\
```

管线不修改原始源文件，所有处理产物输出到 `processed/`。
