## 一、着色器语言与版本

- **语言**：GLSL ES 3.00（WebGL 2.0 对应版本）
- **顶点着色器** `#version 300 es`
- **片元着色器** `#version 300 es`
- **精度**：片元着色器默认 `precision highp float;`

## 二、命名约定

### 2.1 变量前缀

| 前缀 | 作用域 | 示例 |
|------|--------|------|
| `a_` | Attribute（顶点输入） | `a_position`, `a_normal`, `a_uv` |
| `u_` | Uniform（外部输入） | `u_modelMatrix`, `u_viewMatrix`, `u_time` |
| `v_` | Varying（顶点→片元） | `v_worldPos`, `v_normal`, `v_uv` |
| — | 局部变量 | camelCase：`lightDir`, `shadowFactor` |

### 2.2 Uniform Block 命名

```glsl
// ✅ 推荐：语义化命名
uniform Scene {
    mat4 u_viewMatrix;
    mat4 u_projectionMatrix;
    vec3 u_cameraPosition;
    float u_time;
};

uniform Light {
    vec3 u_lightDirection;
    vec3 u_lightColor;
    float u_lightIntensity;
};

// ❌ 禁止：无意义缩写
uniform mat4 V;
uniform mat4 P;
uniform vec3 LP;
```

### 2.3 函数命名

```glsl
// ✅ snake_case 动词先行
vec3 calculateLighting(vec3 normal, vec3 viewDir);
float computeShadow(vec4 worldPos);
vec3 applyBloom(vec3 color, float threshold);

// ❌ 禁止：PascalCase 或无意义命名
vec3 DoLight(vec3 n, vec3 v);
float f1(vec4 p);
```

## 三、渲染通道标准

### 3.1 标准通道定义

| 通道 | 着色器对 | 用途 |
|------|---------|------|
| `depth` | `depth.vert` / `depth.frag` | 阴影贴图生成 |
| `forward` | `standard.vert` / `standard.frag` | 不透明物体前向渲染 |
| `transparent` | `standard.vert` / `transparent.frag` | 透明物体（排序后渲染） |
| `postprocess` | `fullscreen.vert` / `{effect}.frag` | 全屏后处理 |
| `skybox` | `skybox.vert` / `skybox.frag` | 天空盒/环境 |
| `particle` | `particle.vert` / `particle.frag` | 粒子系统 |

### 3.2 标准 Uniform 接口

所有主题着色器必须实现以下最少 Uniform 集：

```glsl
// 顶点着色器最少输入
layout(location = 0) in vec3 a_position;   // 位置
layout(location = 1) in vec3 a_normal;     // 法线
layout(location = 2) in vec2 a_uv;         // UV

// 主题无关 Uniform（渲染管线自动注入）
uniform mat4 u_modelMatrix;
uniform mat4 u_viewMatrix;
uniform mat4 u_projectionMatrix;
uniform mat4 u_normalMatrix;
uniform vec3 u_cameraPosition;
uniform float u_time;                      // 自场景加载起的秒数

// 材质 Uniform（从模型材质参数自动注入）
uniform vec4 u_baseColorFactor;
uniform float u_metallicFactor;
uniform float u_roughnessFactor;
uniform sampler2D u_baseColorTexture;
uniform sampler2D u_metallicRoughnessTexture;
uniform sampler2D u_normalTexture;
```

## 四、扩展风格参数集

### 4.1 设计原则

通用着色器底层**无任何风格硬编码**。所有视觉参数通过 manifest 的 `extensions.{themeId}.shaderParams` 传入着色器 UBO：

```glsl
// 通用后处理着色器框架
uniform StyleParams {
    // 所有风格参数在此 block 中
    // 具体字段由主题 manifest 定义，着色器按名称匹配
    float u_bloomThreshold;
    float u_bloomStrength;
    vec3 u_colorTint;
    // ... 扩展字段
};
```

### 4.2 Cyber-Girl-Apartment 风格参数集（验证案例 1）

```jsonc
{
```

### 4.4 双主题验证覆盖矩阵

| 视觉效果维度 | Cyber-Girl 需求 | 古风水墨需求 | 底层兼容结论 |
|-------------|----------------|-------------|-------------|
| 发光/Bloom | ✅ 强霓虹发光 | ❌ 不需要 | Bloom 参数可独立开关 |
| 扫描线/条纹 | ✅ 横向扫描线 | ❌ 不需要（但纸纹是有机条纹） | 条纹类 shader 可复用框架 |
| 色差 | ✅ RGB 偏移 | ❌ 不需要 | 独立开关 |
| 噪点/颗粒 | ✅ 数字噪点 | ✅ 纸纤维颗粒 | 颗粒系统可复用，参数不同 |
| 粒子 | ✅ 数字雨 | ✅ 水墨飞溅 | 粒子系统框架通用 |
| LUT 调色 | ✅ 霓虹调色板 | ✅ 墨色渐变 | LUT 管道通用 |
| 纹理叠加 | ✅ 全息扫描线 | ✅ 宣纸纹理 | 纹理叠加框架通用 |

**结论**：两种风格在底层着色器上没有结构性冲突，参数集完全正交。

## 五、错误处理

### 5.1 编译失败

```typescript
class ShaderCompiler {
  compile(gl: WebGL2RenderingContext, source: string, type: number): WebGLShader | null {
    const shader = gl.createShader(type)!;
    gl.shaderSource(shader, source);
    gl.compileShader(shader);

    if (!gl.getShaderParameter(shader, gl.COMPILE_STATUS)) {
      const log = gl.getShaderInfoLog(shader);
      gl.deleteShader(shader);
      this.logError(type, log);
      return null; // 返回 null，由调用方决定降级
    }
    return shader;
  }
}
```

**规则**：
- 着色器编译失败 → 返回 null，不抛异常
- 调用方使用 fallback shader（如纯色 shader）
- 记录崩溃日志（驱动错误信息 + shader 路径）

### 5.2 编译超时

- 单个 Shader 编译超过 3s → 中止，视为失败
- 防止恶意/过复杂的 Shader 造成 GPU 驱动超时

### 5.3 Link 失败

- Program link 失败 → 记录日志 → 使用 fallback shader program

## 六、性能约束

| 约束项 | 限制 | 说明 |
|--------|------|------|
| 顶点着色器指令数 | ≤ 512 | 保守估计，适配集成显卡 |
| 片元着色器指令数 | ≤ 1024 | — |
| 纹理采样次数（片元） | ≤ 8 | 包含阴影贴图采样 |
| Uniform Block 数量 | ≤ 4 | 含管线自动注入的 2 个 |
| 循环展开 | ≤ 16 次迭代 | 避免动态循环 |
| 谨慎使用 discard | — | 仅在 alpha-cutout 场景（树叶、栅栏、镂空装饰）下允许；常规透明使用 alpha 混合。滥用 discard 会影响 Early-Z 优化。 |

## 七、着色器资源纳入 NV3D 打包

### 7.1 通用着色器（内置在客户端中）

以下着色器随客户端内置，不打包进 NV3D：

| 着色器 | 用途 |
|--------|------|
| `fullscreen.vert` | 全屏后处理 quad |
| `standard.vert/frag` | 默认 PBR 前向渲染 |
| `depth.vert/frag` | 阴影贴图 |
| `skybox.vert/frag` | 天空盒 |
| `particle.vert/frag` | 默认粒子 |

### 7.2 主题专属着色器（打包进 NV3D）

主题可覆盖或扩展着色器，路径在 manifest 中声明：

```json
{
  "resources": {
    "shaders": {
      "custom_postprocess": {
        "path": "shaders/postprocess.glsl",
        "hash": "sha256:..."
      }
    }
  }
}
```

加载时校验：主题专属着色器签名必须匹配 manifest 中的 hash。
