## 一、本地开发环境

### 1.1 启动命令

```bash
# 主应用开发（3D 模块关闭）
npm run tauri:dev

# 启用 3D 模块调试（手动修改 featureFlag.ts）
# WEBGL3D_ENABLED = true → npm run tauri:dev
```

### 1.2 开发配置

- Feature Flag 默认 `false`
- 调试面板通过 `import.meta.env.DEV` 条件加载
- 3D 模块的 Vite HMR 仅影响 `src/webgl3d/` 下的更改
- Shader 热重载：修改 `.glsl` 文件无需重启应用

### 1.3 代码检查

```bash
# 类型检查（commit 前必须过）
npm run typecheck

# 仅检查 3D 模块相关文件（可选）
npx tsc --noEmit -p tsconfig.app.json | grep "src/webgl3d"
```

## 二、生产构建

### 2.1 构建命令

```bash
npm run tauri:build
```

### 2.2 树摇 (Tree-shaking)

3D 模块代码通过以下方式确保在生产构建中可被完全剔除：

| 条件 | 效果 |
|------|------|
| Feature Flag = false + 生产构建 | Vite 将动态 import 的 3D 代码视为 dead code，不打入 bundle |
| 构建工具配置 | 无需额外配置，依赖 Vite 的 `import()` 动态分割 |

### 2.3 分模块构建策略

如果需要发布"不含 3D 模块"的建置版本：

```bash
# 方案 A：通过环境变量控制
VITE_WEBGL3D=false npm run tauri:build

# 方案 B：Vite 配置中 conditionally exclude
# vite.config.ts
define: {
  __WEBGL3D_ENABLED__: JSON.stringify(process.env.VITE_WEBGL3D !== "false"),
}
```

构建产物对比：

| 构建类型 | 包含内容 | 包体增量 |
|---------|---------|---------|
| 纯净版（VITE_WEBGL3D=false） | 主应用 + 现有功能 | 0 |
| 完整版 | 主应用 + 3D 模块 | + 3D 渲染库（如 Three.js ~300KB gzip（核心 + GLTFLoader + DRACOLoader + OrbitControls + 常用后处理））+ WebGL 自定义代码 |

### 2.4 构建校验

```bash
# 验证 3D 模块在纯净构建中不存在
grep -r "WEBGL3D" dist/  # 预期：无匹配

# 验证完整构建中 3D 模块可被加载
# 手动测试 Feature Flag 开启后的加载流程
```

## 三、服务端 NV3D 元数据 API

### 3.1 API 端点

| 端点 | 方法 | 用途 |
|------|------|------|
| `/api/themes/list` | GET | 获取可用主题列表（含 3D） |
| `/api/themes/{themeId}` | GET | 获取单个主题元数据 |
| `/api/themes/{themeId}/download` | GET | 下载 NV3D 文件（支持 Range） |
| `/api/themes/check-update` | POST | 批量检查主题更新 |

### 3.2 `/api/themes/list` 响应

```json
{
  "themes": [
    {
      "themeId": "cyber-girl-apartment",
      "themeType": "webgl3d",
      "name": { "zh": "赛博公寓", "en": "Cyber Apartment" },
      "version": "1.0.0",
      "nv3dVersion": "2.0",
      "fileSize": 524288000,
      "fileHash": "sha256:abc123...",
      "downloadUrl": "/api/themes/cyber-girl-apartment/download",
      "previewImages": {
        "thumbnail": "https://scm-think.cn/themes/webgl3d/cyber-girl-apartment/thumbnail.webp",
        "hero": "https://scm-think.cn/themes/webgl3d/cyber-girl-apartment/hero.webp"
      },
      "requiredMembership": "member",
      "minAppVersion": "2.0.0"
    }
  ]
}
```

### 3.3 权限过滤

服务端根据请求中的 JWT 判断用户等级，过滤返回结果：
- Free 用户：返回空 `themes: []`
- Member 用户：返回 Member 等级的 3D 主题列表

## 四、CDN 分发

### 4.1 文件路径

```
https://scm-think.cn/themes/webgl3d/
├── cyber-girl-apartment/
│   ├── cyber-girl-apartment_v1.0.0.nv3d
│   ├── thumbnail.webp
│   └── hero.webp
└── ink-wash/
    ├── ink-wash_v1.0.0.nv3d
    ├── thumbnail.webp
    └── hero.webp
```

### 4.2 上传命令

```bash
# 上传 NV3D 文件
scp -i <SSH_KEY_PATH> \
  scripts/webgl3d/output/cyber-girl-apartment_v1.0.0.nv3d \
  <ECS_HOST>:/var/www/themes/webgl3d/cyber-girl-apartment/

# 上传预览图
scp -i <SSH_KEY_PATH> \
  D:\nova-themes-assets\webgl3d\cyber-girl-apartment\preview\* \
  <ECS_HOST>:/var/www/themes/webgl3d/cyber-girl-apartment/
```

### 4.3 CDN 配置

- 启用 HTTP Range 支持（断点续传）
- 启用 gzip/brotli 压缩（对于 `store` 类型的 Block 无效，但对文本类资源有效）
- Cache-Control：主题文件 `max-age=31536000`（1 年，版本号更新即失效）
- 大文件分片下载建议使用阿里云 CDN 或 OSS

## 五、断点续传

### 5.1 客户端下载流程

```
1. 请求 HEAD /api/themes/{themeId}/download
   → 获取 Content-Length, Accept-Ranges: bytes

2. 检查本地是否有未完成的下载
   → 有 → 读取已下载字节数 → 从断点继续
   → 无 → 全新下载

3. 发送 GET with Range header
   Range: bytes={downloaded}-

4. 接收 206 Partial Content
   → 追加写入本地文件

5. 下载完成 → 校验 SHA256
   → 匹配 → 标记完成
   → 不匹配 → 删除 → 重新下载
```

### 5.2 下载进度持久化

```
{app_data_dir}/webgl3d_downloads/
└── {theme_id}.download
    ├── {theme_id}.partial            ← 未完成的下载文件
    └── {theme_id}.meta               ← 下载元信息
        { totalSize, downloadedSize, lastByte, url, startedAt }
```

重启应用后恢复未完成的下载。

## 六、主题增量更新

### 6.1 更新检测

```
客户端请求
POST /api/themes/check-update
Body: { "themes": [{ "themeId": "cyber-girl-apartment", "version": "1.0.0" }] }

服务端响应
{
  "updates": [
    {
      "themeId": "cyber-girl-apartment",
      "currentVersion": "1.0.1",
      "fileSize": 524300000,
      "fileHash": "sha256:newhash...",
      "changedResources": ["models/scene_hd.glb", "textures/scene/diffuse.ktx2"]
    }
  ]
}
```

### 6.2 增量下载

当服务端支持增量更新时：

```
1. 客户端比较新旧 Manifest
2. 仅下载 hash 变更的 Resource Block（而非整个 NV3D 文件）
3. 替换本地缓存中的变更 Block
4. 更新 manifest.json + 版本号
```

### 6.3 全量降级

当增量信息不可用时（如跨多个版本跳升），回退到全量下载。

## 八、分模块构建与发布

### 8.1 发布场景

| 场景 | 构建配置 | 说明 |
|------|---------|------|
| 稳定版（不含 3D） | `VITE_WEBGL3D=false` | 主应用独立发布，零影响 |
| 预览版（含 3D Beta） | `VITE_WEBGL3D=true, FEATURE_FLAG=false` | 3D 代码内置但默认关闭，需手动开启 |
| 正式版（含 3D） | `VITE_WEBGL3D=true, FEATURE_FLAG=true` | 3D 功能对 Member 用户开放 |

### 8.2 版本路线

```
v2.0.0 — 主应用（不含 3D）— 稳定版
v2.1.0-beta — 主应用 + 3D 模块（Feature Flag 默认 false）— 灰度
v2.2.0 — 主应用 + 3D 模块（Feature Flag 默认 Member 可见）— 正式版
```

## 九、CI/CD 集成

### 9.1 构建流水线

```
push to master
    │
    ├─ npm run typecheck
    ├─ npm run lint
    ├─ npm run test
    │
    ├─ 3D 模块检查（如有变更）
    │   ├─ Feature Flag 确认为 false
    │   ├─ 无静态 import 指向主应用
    │   └─ 素材校验脚本通过
    │
    ├─ npm run tauri:build
    │
    └─ 构建产物
        ├─ MSI 安装包
        └─ 上传至发布服务器
```

### 9.2 3D 模块专属 CI 检查

```yaml
# .github/workflows/check-3d-module.yml
- name: Verify Feature Flag Off
  run: |
    grep -q 'WEBGL3D_ENABLED = false' src/webgl3d/featureFlag.ts || \
      (echo "ERROR: Feature flag must be false on master" && exit 1)

- name: Verify No Static Import to 3D Module
  run: |
    ! grep -r "from.*webgl3d" src/components/ src/pages/ src/hooks/ src/stores/ || \
      (echo "ERROR: Main app must not statically import from webgl3d" && exit 1)
```
