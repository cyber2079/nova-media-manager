## 二、WebView2 兼容约束

### 2.1 单 Context 原则

**规则**：全局只存在一个 WebGL 2.0 Context。禁止多画布并行渲染。

```typescript
class RenderManager {
  private static instance: RenderManager;
  private gl: WebGL2RenderingContext | null = null;
  private canvas: HTMLCanvasElement | null = null;

  private constructor() {}

  static getInstance(): RenderManager {
    if (!RenderManager.instance) {
      RenderManager.instance = new RenderManager();
    }
    return RenderManager.instance;
  }

  // 创建唯一 context，若已有则抛错
  createContext(canvas: HTMLCanvasElement): WebGL2RenderingContext {
    if (this.gl && !this.gl.isContextLost()) {
      throw new Error("WebGL context already exists. Destroy first.");
    }
    const gl = canvas.getContext("webgl2", {
      alpha: true,
      antialias: true,
      depth: true,
      stencil: false,
      premultipliedAlpha: true,
      preserveDrawingBuffer: false, // 不保留帧缓冲，节省显存
      powerPreference: "high-performance",
    });
    if (!gl) throw new Error("WebGL 2.0 not available");
    this.gl = gl;
    this.canvas = canvas;
    return gl;
  }

  destroyContext(): void {
    if (!this.gl) return;
    const ext = this.gl.getExtension("WEBGL_lose_context");
    ext?.loseContext();           // 主动释放，不依赖 GC
    this.gl = null;
    this.canvas = null;
  }
}
```

### 2.2 WebGL 可用性检测

初始化前执行快速检测：

```
1. canvas.getContext("webgl2") → 非 null？
2. 创建测试 shader → 编译成功？
3. 渲染一帧到小 framebuffer → 无错误？
4. 检测 WebView2 UserAgent → 记录版本
```

检测失败 → 模块标记为 `unsupported`，走静态降级。

### 2.3 间接 GPU 内存监控

由于 WebView2 无法直接获取 GPU 显存使用量，通过以下指标间接估算：

| 指标 | 获取方式 | 告警阈值 |
|------|---------|---------|
| JS Heap | `performance.memory.usedJSHeapSize` | > 80% of heap limit |
| 三角面数 | 渲染循环统计 | > `triangleBudget`（manifest 定义） |
| DrawCall | 渲染循环统计 | > `drawCallBudget` |
| 纹理计数 | ResourceCache 统计 | > 50 张 4K 纹理或等效 |
| 帧时间 | `performance.now()` delta | 连续 > 33ms（< 30fps）超过 3s |

任一指标触发告警 → 进入降级流程（见第九节）。

## 三、Context 被动丢失处理（睡眠/唤醒）

### 3.1 场景描述

用户合上笔记本盖子 / 系统睡眠 / GPU 驱动更新 — WebGL context 丢失。这是**日常场景**，不是罕见边缘情况。

### 3.2 处理流程

```
webglcontextlost 事件触发
        │
        ├─ 1. 暂停渲染循环（不做任何 GL 调用）
        ├─ 2. 标记场景状态快照（当前场景 ID、相机位置、交互状态）
        ├─ 3. 展示遮罩层："正在恢复..."
        │
        ├─ webglcontextrestored 事件触发？
        │   │
        │   ├─ 是 → 4. 重建所有纹理（gl.texImage2D）
        │   │        5. 重建所有 buffer（gl.bufferData）
        │   │        6. 重建所有 shader program
        │   │        7. 恢复场景状态快照
        │   │        8. 恢复渲染循环
        │   │        9. 移除遮罩层
        │   │
        │   └─ 否（30s 超时）
        │          → 降级：静态 2D 预览图
        │          → 30s 后重试创建新 context
        │          → 仍失败 → 恢复原生壁纸
```

### 3.3 恢复时间预算

Context Lost 恢复的总时间必须控制在可接受范围内：

| 阶段 | 时间预算 | 超时行为 |
|------|---------|---------|
| 重建 Shader Program（所有） | ≤ 3s | 超时 → 标记超时的 shader 使用 fallback |
| 重建纹理 + Buffer（所有） | ≤ 2s | 超时 → 按需懒加载，优先恢复首帧可见资源 |
| 总恢复时间（从 lost 事件到恢复渲染） | ≤ 5s | 超时 → 放弃恢复 → 降级到静态 2D 预览图 |

**实现要点**：
```typescript
const RECOVERY_TIMEOUT = 5000; // 总恢复时间预算 5s
let recoveryStartTime = 0;

canvas.addEventListener("webglcontextlost", (event) => {
  event.preventDefault();
  this.pauseRenderLoop();
  this.captureSceneSnapshot();
  this.showRecoveryOverlay();
  recoveryStartTime = performance.now();
  this.startRecoveryTimer(RECOVERY_TIMEOUT);
});

canvas.addEventListener("webglcontextrestored", () => {
  const elapsed = performance.now() - recoveryStartTime;
  if (elapsed > RECOVERY_TIMEOUT) {
    // 已超时，放弃恢复
    this.degradeToStaticPreview();
    return;
  }
  this.rebuildShadersFirst();     // 优先重建 shader program
  this.rebuildTexturesAndBuffers(); // 然后重建纹理和 buffer
  this.restoreSceneSnapshot();
  this.resumeRenderLoop();
  this.hideRecoveryOverlay();
  this.clearRecoveryTimer();
});
```

## 四、资源生命周期管理

### 4.1 资源状态机

```
[UNLOADED] ──load()──→ [LOADING] ──success──→ [LOADED] ──use──→ [ACTIVE]
                            │                      │
                            └──fail──→ [FAILED]    └──dispose()──→ [DISPOSED]
```

### 4.2 资源分类与缓存策略

| 类别 | 示例 | 缓存策略 | 热切换行为 |
|------|------|---------|-----------|
| 通用着色器 | PBR 基础 shader、阴影 shader | 短期缓存（跨主题复用） | 保留 |
| 通用材质 | 基础 PBR 材质球 | 短期缓存 | 保留 |
| 主题专属 shader | 赛博发光 shader、水墨晕染 shader | 按主题加载 | 完整释放 |
| 主题模型 | 场景、角色、道具 | 按主题加载 | 完整释放 |
| 主题贴图 | 场景贴图、角色皮肤 | 按主题加载 | 完整释放 |
| 主题动画 | 角色动画、过场动画 | 按主题加载 | 完整释放 |
| 音效/BGM | 背景音乐、交互音效 | 按主题加载 | 完整释放 |

### 4.3 资源释放顺序（热切换时）

```
1. 停止所有动画播放
2. 移除场景中所有 mesh 对象
3. 释放几何体（dispose geometry: GPU buffer）
4. 释放材质（dispose material）
5. 释放纹理（dispose texture: gl.deleteTexture）
6. 释放主题专属 shader program（gl.deleteProgram）
7. 执行 gl.flush() — 确保 GPU 指令队列清空
8. GC 提示（大型数组置 null）
```

## 五、渲染循环

### 5.1 循环结构

```
requestAnimationFrame(timestamp)
    │
    ├─ 检查：熔断器是否 OPEN？ → 跳过本次渲染
    ├─ 检查：Context 是否 lost？ → 跳过 → 等待恢复流程
    ├─ 检查：应用是否失焦/最小化？ → 跳过（或降帧）
    │
    ├─ deltaTime = timestamp - lastTimestamp
    ├─ 更新阶段
    │   ├─ 动画更新（AnimationController.tick）
    │   ├─ 粒子系统更新
    │   └─ 过渡/补间更新
    │
    ├─ 渲染阶段
    │   ├─ 场景图遍历 + 视锥剔除
    │   ├─ 阴影 pass（如启用）
    │   ├─ 不透明 pass（前向渲染）
    │   ├─ 透明 pass（排序后）
    │   └─ 后处理 pass（如启用）
    │
    ├─ 帧后处理
    │   ├─ 性能指标采集（FPS、帧时间、DrawCall）
    │   └─ 降级检测（连续超帧时 → 触发降级）
    │
    └─ requestAnimationFrame(timestamp) — 下一帧
```

### 5.2 帧预算

| 目标 | 单帧预算 | 说明 |
|------|---------|------|
| 60 FPS | 16.6ms | 默认目标 |
| 30 FPS | 33.3ms | 降帧运行 |
| > 33ms 持续 3s | — | 触发降级评估 |

## 六、功耗管理

### 6.1 渲染频率自适应

| 应用状态 | 目标帧率 | 行为 |
|---------|---------|------|
| 用户活跃交互中 | 60 FPS | 全速渲染 |
| 用户无操作 10s（场景有动画） | 30 FPS | 降帧 |
| 用户无操作 10s（场景无动画） | 跳过渲染 | 仅监听交互事件 |
| 应用失焦/最小化 | 暂停循环 | 不调用 rAF |
| 恢复焦点 | 60 FPS | 立即恢复全速 |

### 6.2 性能模式

设置面板提供用户可选模式：

```
性能模式
├─ 画质优先（60fps，完整后处理）
├─ 均衡（60fps，简化后处理）
└─ 节能（30fps，无后处理）
```

## 七、主题热切换完整生命周期

### 7.1 切换流程

```
用户选择新主题
    │
    ├─ 1. 暂停交互系统
    ├─ 2. 保存当前主题用户存档
    ├─ 3. 展示过渡 UI（见 7.2）
    ├─ 4. 执行资源释放（见 4.3 释放顺序）
    ├─ 5. 打开新 NV3D 文件
    ├─ 6. 读取 Manifest
    ├─ 7. 加载低模预览资源 → 渲染首帧
    ├─ 8. Worker 后台加载高精度资源 → 渐进替换
    ├─ 9. 读取新主题用户存档
    ├─ 10. 恢复交互系统
    └─ 11. 移除过渡 UI
```

### 7.2 过渡 UX

| 切换场景 | 过渡方式 | 最长耗时 |
|---------|---------|---------|
| 共享 shader 缓存命中（快速切换） | 上一主题最后一帧定格 + 0.3s 淡出 → 新主题淡入 | < 1s |
| 需要加载全新 shader（冷切换） | 主题封面图 + 进度条 + 主题名称 | 预计 2-5s |
| 加载超时 | 提示"加载超时"，回退到上一主题或静态预览 | > 10s 触发 |

### 7.3 用户存档隔离

- 切换主题时卸载当前存档，异步读取新主题独立数据表
- 不同主题的存档完全隔离，不互相覆盖
- 存档结构见 [16_用户存档数据结构](16_3D用户存档通用数据结构.md)

## 八、窗口 Resize / DPI 适配

### 8.1 Resize 处理

```
窗口 resize 事件
    ↓
debounce 200ms（防频繁重建）
    ↓
更新 Canvas 尺寸（width/height 属性）
    ↓
更新 gl.viewport
    ↓
更新渲染目标（如需 off-screen render target）
    ↓
更新相机宽高比
    ↓
触发重渲染（非全量资源重建）
```

### 8.2 DPI 变化

```
devicePixelRatio 变化（多显示器切换）
    ↓
更新 Canvas 像素比（width/height × dpr）
    ↓
更新渲染目标分辨率
    ↓
触发重渲染
```

### 8.3 虚拟显示适配器

远程桌面/投屏环境下 WebGL 可能不可用：
- `getContext("webgl2")` 返回 null → 直接走静态降级
- 不尝试软件渲染

## 九、降级策略

### 9.1 降级链路

```
3D 正常渲染（60fps，完整画质）
    │
    ├─ 连续 3s 帧时间 > 33ms（< 30fps）
    │       ↓
    │   阶段 1：降帧（60 → 30fps）
    │       ↓
    ├─ 降帧后仍 > 33ms 持续 5s
    │       ↓
    │   阶段 2：降画质（下調一個 quality level）
    │       ↓
    ├─ 最低画质仍不稳定
    │       ↓
    │   阶段 3：切换为静态 2D 主题预览图
    │       ↓
    ├─ 静态预览图加载失败
    │       ↓
    │   阶段 4：恢复原生壁纸 + 通知用户模块已禁用
    │
    └─ GPU 显存不足/WebGL 完全不可用
            ↓
        直接跳到阶段 4
```

### 9.2 降级恢复

降级后每 60s 尝试恢复一级（如阶段 3 → 阶段 2），共尝试 3 次。3 次均失败则维持当前降级级别。

## 十、性能指标采集

### 10.1 必采指标

| 指标 | 采集频率 | 用途 |
|------|---------|------|
| FPS | 每帧 | 降级判断 |
| 帧时间（ms） | 每帧 | 降级判断 |
| JS Heap（MB） | 每 5s | OOM 预警 |
| DrawCall 数 | 每帧 | 渲染预算监控 |
| 三角面数 | 场景加载时 | 渲染预算监控 |
| 纹理数量 | 资源加载时 | 显存占用估算 |
| 资源加载耗时 | 每个资源加载完成时 | 加载流程优化 |
| Context Lost 次数 | 事件触发时 | 兼容性分析 |

### 10.2 指标使用

- 开发环境 → 显示在 Stats.js 面板
- 生产环境 → 仅用于内部降级判断和崩溃日志，不作为遥测上报
