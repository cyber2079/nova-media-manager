## 一、功能边界定义

### 1.1 3D 主题扩展是什么

3D 主题是基于 WebGL 的交互式桌面场景，用户在软件内导入一个 NV3D 主题包后，可以看到和操作的**完整三维空间**：
- 3D 场景（房间、场景、环境）
- 可交互角色（动画状态机、对话触发）
- 可交互道具（拖拽放置、解锁收集）
- 剧情任务（对话链、解锁条件）
- 动态视觉（粒子系统、后处理特效、光照变化）

### 1.2 3D 主题不是什么

- ❌ 不是独立游戏 — 没有独立启动器、没有存档云同步、没有成就系统
- ❌ 不是壁纸引擎替代 — 不替代原生壁纸功能，两者互斥
- ❌ 不是性能监视器 — 不展示系统信息、不覆盖桌面图标
- ❌ 不是独立窗口 — 嵌入主应用窗口内渲染

## 二、会员权限分层

### 2.1 功能分层表

| 功能 | Free | Member |
|------|:---:|:---:|
| 3D 主题入口可见 | ❌ | ✅ |
| 3D 主题下载 | ❌ | ✅ |
| 3D 主题激活使用 | ❌ | ✅ |
| 3D 主题交互（角色/道具/剧情） | ❌ | ✅ |
| 3D 主题用户存档 | ❌ | ✅ |
| 主题自动更新 | ❌ | ✅ |
| 自定义着色器参数 | ❌ | ✅ |

### 2.2 Free 用户体验

Free 用户默认不感知 3D 主题功能的存在：
- 设置面板不显示 3D 主题相关选项
- 主界面不显示 3D 主题入口
- 不发起 3D 主题相关的网络请求
- 已下载的 3D 主题文件（如从 Member 降级而来）保留但不加载

### 2.3 权限状态感知

前端 3D 组件在渲染前必须检查权限：

```typescript
// 伪代码
function ThreeDEntry() {
  const isMember = useGate("premium-theme");
  if (!isMember) return null; // Free 用户完全看不到
  return <ThreeDCanvas />;
}
```

### 2.4 会员过期处理

- 过期当日 → 3D 模块自动禁用，当前场景保留最后一帧静态图
- 已下载的 NV3D 文件保留在本地（不删除）
- 用户存档保留在独立数据表（不删除）
- 续费后 → 自动恢复，模块重新启用，存档可继续使用

## 三、3D 主题与原生壁纸互斥规则

### 3.1 运行互斥

3D 主题画布与原生壁纸层**不可同时激活**。

| 场景 | 行为 |
|------|------|
| 用户开启 3D 主题 | 自动禁用原生壁纸（图片/视频/幻灯片），壁纸设置保留不丢失 |
| 用户关闭 3D 主题 | 自动恢复壁纸至关闭前的设置 |
| 3D 模块崩溃 | 自动恢复壁纸，不黑屏 |
| 3D 模块加载中 | 壁纸保持显示，3D 就绪后切换 |
| 3D 降级为静态预览 | 壁纸保持禁用，展示静态预览图 |

### 3.2 开关入口

软件设置面板提供独立开关：

```
设置 → 主题 → 3D 主题
  ├─ [开关] 启用 3D 主题
  ├─ 当前主题：Cyber-Girl-Apartment
  ├─ [按钮] 切换主题
  └─ [按钮] 管理已下载主题
```

### 3.3 故障自动恢复

```
3D 模块运行中
    │
    ├─ 渲染正常 → 壁纸保持禁用
    │
    ├─ Context Lost（睡眠唤醒等）→ 尝试重建 context
    │   ├─ 成功 → 继续渲染
    │   └─ 失败 → 降级静态图 → 30s 后重试 → 仍失败 → 恢复壁纸
    │
    ├─ 连续 3 次异常 → 自动禁用模块 → 恢复壁纸 → 通知用户
    │
    └─ 用户手动关闭开关 → 恢复壁纸 → 卸载模块
```

## 四、多主题管理

### 4.1 主题状态模型

一个 3D 主题在客户端有以下状态之一：

```
[未拥有] → (购买/激活) → [可下载]
                              ↓
                         [下载中]
                              ↓
                         [已下载·未激活]
                              ↓
                         [已下载·已激活]  ← 同时只能有一个
                              ↓
                         (会员过期)
                              ↓
                         [已下载·已锁定]  ← 不删除，不能激活
                              ↓
                         (续费)
                              ↓
                         [已下载·已激活]  ← 自动恢复
```

### 4.2 激活规则

- 同一时间只能激活 **1** 个 3D 主题
- 切换主题时执行完整资源清理（见 [05_渲染管线](05_3D场景通用渲染管线规范.md) 热切换章节）
- 用户存档按主题隔离，切换后读取新主题的独立存档

### 4.3 下载管理

- 支持断点续传（HTTP Range 请求）
- 下载进度展示（百分比 + 剩余时间估算）
- 下载失败自动重试 3 次，间隔递增
- 下载完成校验 SHA256，校验失败提示重新下载

### 4.4 主题删除

- 用户可删除未激活的 3D 主题
- 删除操作清理：NV3D 文件 + 解压缓存 + 用户存档（需用户确认）
- 已激活主题不可删除（需先切换到其他主题或关闭 3D 模块）

### 4.5 本地缓存策略

- 解压缓存存放于临时目录，LRU 淘汰
- 用户可配置缓存容量上限（默认 2GB）
- 超过上限 → 自动清理最久未使用的主题缓存
- 当前激活主题的缓存不被淘汰

## 五、功能启停与降级规则

### 5.1 模块启用流程

```
用户点击"启用 3D 主题"
    ↓
检查：当前是否为 Member？ → 否 → 提示升级
    ↓ 是
检查：是否有已下载的 3D 主题？ → 否 → 引导下载
    ↓ 是
禁用原生壁纸
    ↓
加载 NV3D → 读 Manifest → 渲染低模预览
    ↓
后台加载高精度资源 → 渐进替换
    ↓
进入可交互状态
```

### 5.2 模块停用流程

```
用户点击"关闭 3D 主题"
    ↓
暂停渲染循环
    ↓
保存当前用户存档
    ↓
执行全资源 dispose（见 05 文档热切换章节）
    ↓
卸载 WebGL context
    ↓
恢复原生壁纸
    ↓
3D 模块代码不再占用内存
```

### 5.3 自动降级链

```
3D 正常 → [连续 N 次渲染帧超时]
           ↓
         降帧运行（60→30fps）
           ↓ [fps 仍低于 15]
         降低画质（关闭后处理、降低纹理分辨率）
           ↓ [仍不稳定]
         切换为静态 2D 主题预览图
           ↓ [预览图加载失败]
         恢复原生壁纸 + 通知用户模块已禁用
```

### 5.4 独立开关的持久化

模块开关状态写入 `settings` 表，与主设置一起持久化：
- 重启软件后保持上次的开关状态
- 若上次为异常降级禁用 → 重启后保持禁用，需用户手动重新开启

## 六、NV3D 主题文件使用规范

### 6.1 导入方式

- 用户在设置面板点击"导入主题"
- 系统文件选择器过滤 `.nv3d` 文件
- 校验 Ed25519 签名和 SHA256 完整性 → 校验通过后纳入管理
- 不验证则拒绝导入并提示"文件无效或不完整"

### 6.2 分发约束

- 所有 3D 主题统一封装为 NV3D 单文件分发
- 不支持零散 3D 资源的导入
- 不支持用户自行修改 NV3D 内容后重新打包

### 6.3 版本更新

- 服务端发布新版本 → 客户端收到更新通知
- 更新下载对比 Manifest，仅下载变更的 Resource Block
- 更新后用户存档自动迁移（如有 Schema 变更）
