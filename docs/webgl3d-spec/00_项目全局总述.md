## 一、基础事实提要

1. 主体软件为成熟桌面客户端，具备完整运行基座、本地文件管理、会员付费、授权加密、本地数据库、专属资源打包加密体系。本次仅新增内置扩展模块，不重构原有软件主体、底层基座、现有业务逻辑。

2. 开发内容：一套通用可复用 WebGL 交互式 3D 主题扩展功能，归属软件增值模块，非独立项目、独立游戏。底层架构、渲染管线、素材流水线、代码规范全局通用，可支撑后续全部 3D 主题迭代。**Cyber-Girl-Apartment（赛博公寓）为本模块首个落地验证案例**，**古风水墨为第二个同步验证案例**。所有底层逻辑不绑定单一主题。

3. 交付产物标准：全部 3D 主题资源、配置、元数据统一打包生成应用专属 NV3D 主题文件，终端仅需导入加载即可完整运行整套交互主题，无需额外拆分素材、手动配置参数。

4. 素材生产规则：全程本地离线工作流，采用开源本地 3D/贴图生成工具，禁止云端第三方 3D 生成服务，规避商用资产泄露风险。

5. 基座兼容要求：扩展模块完整复用软件现有全部底层基座能力，新增代码、接口、存储逻辑自动适配现有技术体系，无需额外新增独立运行环境、第三方底层框架。依托现有工程文件、依赖配置推导适配当前软件技术底座，文档仅作推荐参考，不强制绑定具体技术版本。关键接口和行为约束以本文档为准，实现时可选用等效替代方案。

6. 隔离开发硬性要求：全部新增功能采用完全分层隔离方案，代码、资源、接口、存储目录独立划分，与项目原有业务代码解耦；模块具备独立开关、故障熔断机制。模块异常、开发调试崩溃、逻辑缺陷时，不会阻塞、破坏主程序原有全部功能，保障主软件可正常独立运行，模块可一键禁用隔离。

## 三、开发红线禁止项

1. 禁止将 3D 扩展模块设计为独立游戏架构，必须深度嵌入现有软件体系，同时保持代码完全解耦隔离；
2. 禁止引入云端第三方 3D 生成服务相关流程与代码；
3. 禁止硬编码单一案例主题逻辑，底层代码需通过 **Lockstep 双主题**（Cyber-Girl-Apartment + 古风水墨）验证方能冻结；
4. 禁止改动、重构软件原有授权、加密、本地存储、付费权限底层逻辑，新功能仅做复用扩展；
5. 禁止产出存在性能缺陷、资源泄漏、规格不达标代码与素材方案；
6. 禁止文档、代码内固化、指定底层技术框架、版本，依托现有工程自动适配基座；
7. 禁止直接修改主程序原有核心逻辑、全局入口、基础工具类、核心数据表；所有交互采用封装中转接口调用，新增数据独立分表、新增资源独立目录；
8. 禁止模块异常无兜底处理，必须增加熔断、降级、关闭开关，模块故障时自动卸载，不占用主线程、不阻塞主界面、不破坏本地原有用户数据；
9. 禁止分散分发零散模型、贴图、配置文件，所有主题内容必须统一封装为标准 NV3D 文件对外交付使用；
10. 禁止软件渲染 3D（SwiftShader/WARP），降级路径一律走静态 2D 主题预览图，不做低帧率 3D 幻灯片。

## 四、核心架构决策

### 4.1 NV3D 容器格式

**决策**：不修补 ZIP 格式，独立设计 NV3D 容器格式。

**理由**：ZIP 不适合 3D 资源的随机读取需求——无原生索引、整体压缩对已压缩格式（Draco、KTX2）无效。修补 ZIP 不如从根上设计对的格式。

**格式结构**：
```
┌─────────────────────────────────┐
│  Magic Bytes: "NV3D" (4B)       │
├─────────────────────────────────┤
│  Header (64B)                   │
├─────────────────────────────────┤
│  Manifest Index (JSON, gzip)     │
├─────────────────────────────────┤
│  Resource Block 1               │  ← AES-256-GCM 加密
│  Resource Block 2               │
│  ...                            │
├─────────────────────────────────┤
│  Footer (96B, 含 Ed25519 签名)   │
└─────────────────────────────────┘
```

关键特性：
- Manifest 在 Header 之后立即出现，无需尾部定位
- 每个 Resource Block 独立压缩/SHA256，损坏时定位到具体资源
- AES-256-GCM 加密（flags 控制）
- Ed25519 签名（Footer）

详见 [08_加密3D资源加载通用流程](08_加密3D资源加载通用流程.md) 和 [17_专属资源包打包加密通用规范](17_专属资源包打包加密通用规范.md)。

### 4.2 Lockstep Development（双主题同步开发）

**决策**：Cyber-Girl-Apartment 和古风水墨两个主题同步开发、互相验证。

**理由**：单一案例的隐性假设会沉淀进底层抽象。两个风格截然不同的主题同步推进，发现问题时底层接口即刻调整，而非事后补救。

| 底层能力 | Cyber-Girl 验证 | 古风水墨验证 |
|---------|----------------|-------------|
| 着色器参数集 | 霓虹发光、扫描线、色差 | 墨色晕染、宣纸纹理、留白 |
| 粒子系统 | 数字雨、全息粒子 | 水墨飞溅、烟云飘散 |
| 后处理 | Bloom + 色差 + 噪点 | 边缘柔化 + 纸纹叠加 |
| 动画状态机 | 全息投影出场 | 卷轴展开出场 |
| 交互模式 | 点击全息面板 | 点击画卷/折扇 |
| 光照模型 | 自发光 + 点光源主导 | 环境光 + 漫反射主导 |

**执行规则**：Cyber-Girl 每实现一个底层能力，古风水墨同一天内写出对应的配置/skeleton。古风水墨用现有底层接口实现不了 → 当天调整底层接口。

详见 [18_WebGL 3D主题标准化开发流程](18_WebGL%203D主题标准化开发流程.md)。

**执行策略**：Cyber-Girl-Apartment 做完整实现（首个落地验证案例），古风水墨做骨架验证（极简场景 + 关键 shader 参数配置 + 1-2 个核心交互）。古风水墨用现有底层接口实现不了 → 当天调整底层接口。两个主题的素材工作量不对等——骨架验证不要求完整素材产出。

### 4.3 阶段 0：可行性验证

**决策**：在编写任何规范文档之前，先完成 WebView2 WebGL 性能基准测试。

**验证内容**：
- 构建压力场景：10 万三角面 + 多张贴图 + 一套后处理 pass
- 实测数据：FPS、帧时间（16.6ms 预算）、JS heap 波动、context 创建/销毁耗时
- 产出：性能基准数据 → 决定 05/19 文档中的三角面上限、纹理总尺寸、DrawCall 红线

**验证失败的后备方案**：如果 WebView2 WebGL 性能无法满足 60fps 底线，重新评估技术路线（如考虑用 Tauri 的 custom protocol 注册一个原生的 GPU 渲染窗口）。

详见 [18_WebGL 3D主题标准化开发流程](18_WebGL%203D主题标准化开发流程.md)。

## 五、六层纵深防线（运行时隔离体系）

模块异常不扩散不依赖开发者自觉，靠六层强制机制：

```
层级 1 — 编译层：动态 import + Feature Flag
  │  主应用无静态 import 指向 3D 模块
  │  3D 代码语法错误不影响主应用编译
  │  Feature Flag 为 false 时 3D 代码 0 字节进入主线程
  │
层级 2 — 加载层：懒加载 + try-catch
  │  import() 失败 → catch → 标记 disabled → 记录日志
  │  主应用继续运行
  │
层级 3 — 渲染层：React Error Boundary
  │  WebGL context lost / shader 编译失败 / 渲染循环异常
  │  → Error Boundary 捕获 → 卸载画布 → 渲染静态降级内容
  │
层级 4 — 线程层：Web Worker
  │  资源解压、Draco 解码、贴图预处理在 Worker 执行
  │  Worker 崩溃 → 主线程收到 error 事件 → 记录日志
  │
层级 5 — 数据层：独立数据表
  │  3D 模块读写独立 SQLite 表
  │  迁移失败不影响现有表
  │
层级 6 — 运行时层：熔断 + 自动降级
  │  连续 N 次异常 → 自动禁用 3D 模块
  │  降级路径：3D 不可用 → 静态 2D 主题预览图 → 原生壁纸
```

每一层是独立防线，前一层能拦住的问题不会被漏到下一层。

## 六、遗漏补齐清单（9 项）

以下为第一轮架构方案中遗漏、经全面审查后补齐的关键项：

### 6.1 🔴 睡眠/唤醒 → WebGL Context 被动丢失

**问题**：用户合盖再打开，WebGL context 几乎一定 lost。这比崩溃更常见——是日常场景。

**方案**：
- 监听 `webglcontextlost` / `webglcontextrestored` 事件
- 恢复策略：尝试重建 context → 按需重新加载资源 → 恢复到丢失前场景状态
- 恢复失败：降级到静态预览图

**挂载**：[05_渲染管线规范](05_3D场景通用渲染管线规范.md) — 独立章节

### 6.2 🟢 NV3D 与普通主题的独立性

**问题**：NV3D 与现有 NVTP 普通主题是否需要兼容？

**决策**：不需要。NV3D 使用独立魔数 "NV3D" 和独立扩展名 .nv3d，旧客户端识别为不支持的文件类型。普通主题继续使用 .nvtp + NVTP 魔数。两套格式完全独立，互不干扰。

**挂载**：08、17

### 6.3 🟡 主题安全边界 — 恶意 NV3D 文件

**问题**：NV3D 含 GLSL shader 和 JSON 配置。恶意 shader 可触发 GPU 无限循环 → TDR（Timeout Detection & Recovery）→ 桌面闪烁。恶意 JSON 若被不当处理可导致 XSS。

**方案**：
- NV3D 文件必须签名验证通过才加载（复用现有加密体系）
- Shader 编译超时检测（编译超过阈值视为失败）
- 元数据解析走严格 JSON Schema 校验，禁用 `eval` 和 `dangerouslySetInnerHTML`

**挂载**：[08_加密资源加载](08_加密3D资源加载通用流程.md) — 安全校验章节

### 6.4 🟡 Free 用户的 3D 主题体验

**问题**：Member 功能，Free 用户的 UX 是什么？

**方案**：三个选项，根据产品策略选择或组合：
- A：完全不显示 3D 主题入口（最省事，Free 用户无感知）
- B：展示 3D 主题预览，点击提示升级（创造转化压力）
- C：首次安装 7 天试用（体验驱动转化）

前端 UI 需感知权限状态，3D 相关组件在 Free 用户侧不渲染。

**挂载**：[01_产品通用规范](01_产品通用规范.md) — 权限分层章节

### 6.5 🟡 Canvas 与 React UI 层的 Z 序和事件穿透

**问题**：3D 画布是全屏层还是嵌入区域？画布上的鼠标事件哪些属于 3D 交互、哪些穿透给 React UI？

**方案**：
- 3D 画布默认作为全屏背景层，z-index 在 UI 层之下
- 交互区域通过 manifest 声明，非交互区域自动穿透
- UI 面板打开时，画布暂停交互事件接收

**挂载**：[12_画布组件](12_WebGL画布通用组件开发规范.md) + [07_交互系统](07_3D交互系统通用设计标准.md)

### 6.6 🟢 持续渲染的功耗问题

**问题**：60fps WebGL 持续渲染 = 笔记本风扇常转 + 电池雪崩。

**方案**：
- 场景静止（无动画、无交互）时降至 30fps 或跳过帧
- 应用失焦/最小化 → 暂停 requestAnimationFrame 循环
- 可选"性能模式"：30fps / 60fps / 自适应
- 节能策略仅影响 3D 模块，不影响主程序

**挂载**：[05_渲染管线](05_3D场景通用渲染管线规范.md) + [19_性能测试](19_3D渲染通用性能测试与优化标准.md)

### 6.7 🟢 窗口 Resize / DPI 变化

**问题**：用户拖拽窗口、多显示器混合 DPI、远程桌面/投屏虚拟适配器等场景下 WebGL 可能出问题。

**方案**：
- Resize → Canvas 像素比更新 → render target 重建（debounce 处理防频繁重建）
- DPI 变化 → 自动适配 devicePixelRatio
- 虚拟显示适配器无 WebGL → 直接走静态降级

**挂载**：[05_渲染管线](05_3D场景通用渲染管线规范.md) + [12_画布组件](12_WebGL画布通用组件开发规范.md)

### 6.8 🟢 多主题的下载/存储生命周期

**问题**：用户可能同时拥有多个已下载/未下载/已过期的 3D 主题，需要管理完整的生命周期。

**方案**：
- 主题状态：已下载已激活 / 已下载未激活 / 可下载未下载 / 会员过期已锁定
- 本地缓存 LRU 淘汰，用户可配容量上限
- 会员过期后已下载主题锁定但不删除
- 重装软件后恢复已购买主题的下载权限

**挂载**：[01_产品通用规范](01_产品通用规范.md) — 主题生命周期章节

### 6.9 🟢 服务端下载配合

**问题**：NV3D 3D 主题文件数百 MB，需要服务端分发能力。

**方案**：
- HTTP Range 请求支持断点续传
- 主题版本更新：manifest 对比 → 增量下载变更的 Resource Block
- 服务端元数据 API：客户端拉取可下载主题列表，匹配用户权限
- CDN 分发（阿里云 CDN）

**挂载**：[20_构建部署规范](20_客户端与服务端通用构建部署规范.md)

## 七、Feature Flag：全局功能开关

```typescript
// src/webgl3d/featureFlag.ts
export const WEBGL3D_ENABLED = false; // 开发阶段默认 false
```

**行为规则**：
- `false`：主应用不执行 `import("./webgl3d/...")`，3D 模块 0 代码进入主线程
- 开发调试时开发者手动设为 `true`，提交代码前必须恢复为 `false`
- Pre-commit hook 强制检查：若 flag 为 `true` 则阻断提交
- 生产环境通过设置面板开关控制

详见 [02_全局开发强制标准](02_全局开发强制标准.md)。
