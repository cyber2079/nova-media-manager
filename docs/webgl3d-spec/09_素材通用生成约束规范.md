## 一、模型规范

### 1.1 格式与版本

| 参数 | 要求 |
|------|------|
| 文件格式 | glTF 2.0（`.gltf` + `.bin`）或 GLB（`.glb`） |
| 坐标系统 | 右手坐标系，Y 轴向上，Z 轴向前 |
| 单位 | 1 单位 = 1 米 |
| 模型中心 | 放置于原点 (0, 0, 0) |

### 1.2 面数预算

| 模型类别 | 面数上限（三角面） | 说明 |
|---------|------------------|------|
| 完整场景（HD） | 100,000 | 一个场景的全部模型总和 |
| 完整场景（Low） | 15,000 | 低精度预览版 |
| 单个角色 | 20,000 | 含服装、配饰 |
| 单个道具 | 5,000 | 可交互道具 |
| 装饰道具 | 1,000 | 不可交互的背景装饰 |

### 1.3 模型结构

```jsonc
// glTF 节点层级要求
{
  "nodes": [
    { "name": "Root", "children": [1, 2, 3] },
    { "name": "Architecture", "children": [...] },   // 建筑结构（不可移动）
    { "name": "Furniture", "children": [...] },      // 家具（部分可交互）
    { "name": "Props", "children": [...] },          // 道具（可拖拽）
    { "name": "Characters", "children": [...] }      // 角色（如有）
  ]
}
```

- 节点命名使用 PascalCase，语义化（如 `Door_Front`、`Desk_Main`）
- 交互物体在 glTF `extras` 中标记 `interactable: true`
- 提供 LOD（Level of Detail）：HD + Low 至少两档

### 1.4 压缩

| 压缩方式 | 适用 | 参数 |
|---------|------|------|
| Draco | 几何体数据 | compressionLevel: 7-10 |
| meshopt | 几何体数据（备选） | — |

- 打包前执行压缩
- 压缩后文件关联 manifest 中 `compression: "draco"`

## 二、贴图规范

### 2.1 格式

| 贴图类型 | 推荐格式 | 原因 |
|---------|---------|------|
| Base Color / Diffuse | KTX2 (Basis UASTC) | GPU 原生，体积小 |
| Normal | KTX2 (Basis ETC1S) | 法线细节保留 |
| Metallic-Roughness | PNG / KTX2 | — |
| Emissive | KTX2 | 自发光贴图 |
| Occlusion | PNG (灰度) | 可合并到 MR 贴图 |
| Skybox / Environment | KTX2 (Cube) | — |

### 2.2 分辨率限制

| 贴图类别 | 最大分辨率 | 说明 |
|---------|-----------|------|
| 场景主贴图 | 4096×4096 | diffuse / normal |
| 场景次要贴图 | 2048×2048 | roughness / ao |
| 角色贴图 | 2048×2048 | 全套 |
| 道具贴图 | 1024×1024 | — |
| 粒子贴图 | 512×512 | — |
| 预览缩略图 | 256×256 | thumbnail.webp |
| 预览主图 | 1920×1080 | hero.webp |

### 2.3 材质通道

遵循 glTF PBR Metallic-Roughness 模型：

| 通道 | 必需 | 说明 |
|------|:---:|------|
| Base Color | ✅ | sRGB 色彩空间 |
| Metallic-Roughness | ✅ | R=遮挡(可选), G=粗糙度, B=金属度 |
| Normal (Tangent Space) | ✅ | OpenGL 标准（Y+ 向上） |
| Emissive | ❌ | 自发光，主题风格决定 |
| Occlusion | ❌ | 可与 MR 贴图 R 通道合并 |

### 2.4 UV 要求

- 必须提供 UV0（用于 Base Color + Normal）
- 光照贴图用 UV1（如需烘焙全局光照）
- UV 不翻转、不重叠（除对称贴图外）

## 三、动画规范

### 3.1 骨骼动画

| 参数 | 要求 |
|------|------|
| 骨骼数量 | ≤ 128（保守值推荐 ≤ 64，主题按需选择） |
| 动画格式 | glTF 内嵌或独立 `.glb` |
| 帧率 | 30 FPS |
| 动画时长 | 单段 ≤ 30 秒 |

### 3.2 动画命名

```
{characterId}_{action}[_{variant}]

示例：
cyber_girl_idle
cyber_girl_greet
cyber_girl_walk
cyber_girl_interact_01
```

### 3.3 根骨骼约定

- 所有角色统一使用 `Hips` / `mixamorig:Hips` 作为根骨骼
- 动画控制器通过骨骼名称匹配（不依赖骨骼索引）

## 四、文件命名强制标准

### 4.1 通用规则

- **全部小写**，单词用下划线 `_` 连接（snake_case）
- 不包含空格、中文、特殊字符（除 `_` 和 `.`）
- 扩展名全小写

### 4.2 按文件类型

| 类型 | 命名格式 | 示例 |
|------|---------|------|
| 模型 | `{object}_{quality}.glb` | `scene_low.glb`, `scene_hd.glb`, `character_cyber.glb` |
| 贴图 | `{object}_{type}.ktx2` | `scene_diffuse.ktx2`, `scene_normal.ktx2` |
| 动画 | `{character}_{action}.glb` | `cyber_girl_idle.glb`, `cyber_girl_greet.glb` |
| 着色器 | `{effect}.glsl` | `postprocess.glsl`, `custom_bloom.glsl` |
| 音频 | `{type}_{id}.ogg` | `bgm_main.ogg`, `sfx_door_open.ogg` |
| i18n | `{lang}.json` | `zh.json`, `en.json` |
| 预览图 | `{size}.webp` | `thumbnail.webp`, `hero.webp` |

### 4.3 错误示例

```
❌ Scene_High.glb          — 含大写
❌ scene high.glb          — 含空格
❌ 场景_高精度.glb          — 含中文
❌ scene-high#final.glb    — 含特殊字符
✅ scene_hd.glb            — 正确
```

## 五、Cyber-Girl-Apartment 风格参数案例

### 5.1 视觉风格定义

| 参数 | 值 |
|------|-----|
| 风格 | 赛博朋克 |
| 色调 | 暗冷基底（深蓝/紫黑）+ 霓虹高亮（品红/青色/橙） |
| 材质 | 金属 + 玻璃 + 全息投影 + 发光面板 |
| 光照 | 点光源为主，环境光极暗 |
| 后处理 | Bloom 强、扫描线、色差 |

### 5.2 模型面数预算

| 模型 | 預估面数 |
|------|---------|
| 公寓房间（墙/地板/天花板） | 5,000 |
| 家具（桌椅/柜子/床） | 8,000 |
| 电子设备（屏幕/投影仪/控制面板） | 4,000 |
| 装饰（霓虹灯/海报/线缆） | 3,000 |
| 全息投影粒子区域 | 5,000（粒子不占模型面数） |
| 角色（赛博女孩） | 18,000 |

### 5.3 贴图清单（预估）

| 贴图 | 分辨率 | 说明 |
|------|--------|------|
| 房间 diffuse + normal | 4096×4096 | 墙面/地板纹理 |
| 家具 diffuse + normal | 2048×2048 | 金属/塑料材质 |
| 霓虹灯 emissive | 2048×2048 | 发光面板 |
| 屏幕内容 diffuse | 2048×2048 | 屏幕静态内容 |
| 角色全套 | 2048×2048 | 皮肤/服装/头发 |
| 全息贴图 | 512×512 | 粒子贴图 |
