## 二、射线拾取（RayPicker）

### 2.1 拾取管线

```
1. 屏幕坐标 → NDC 坐标转换
2. 相机矩阵求逆 → 世界空间射线
3. 场景图遍历 → AABB 粗筛
4. 三角面级精确求交 → 命中列表
5. 按距离排序 → 返回最近命中
```

### 2.2 可拾取对象标记

```jsonc
// Mesh 在 glTF 的 extras 中声明可拾取属性
{
  "extras": {
    "interactable": true,
    "interactionId": "click_door",
    "pickShape": "mesh",        // "mesh" | "sphere" | "box" — 拾取几何体类型
    "highlightOnHover": true,
    "highlightColor": [1.0, 0.5, 0.0]
  }
}
```

### 2.3 拾取优先级

```
1. UI 遮罩层 → 拦截所有拾取（如设置面板打开时）
2. 拖拽中的道具 → 独占拾取
3. 可交互物体 → 按距离近到远
4. 场景背景 → 相机旋转/缩放操作
```

## 三、动画状态机

### 3.1 状态机模型

```
         ┌──────────┐
    ┌───→│  idle    │←───┐
    │    └────┬─────┘    │
    │         │          │
    │    [交互触发]      │
    │         │          │
    │         ▼          │
    │    ┌──────────┐    │
    │    │ 动画_01   │    │
    │    └────┬─────┘    │
    │         │          │
    │    [动画结束]      │
    │    (nextAnim)      │
    │         │          │
    │         ▼          │
    │    ┌──────────┐    │
    └────│ 过渡回 idle│───┘
         └──────────┘
```

### 3.2 状态定义

```jsonc
{
  "animations": {
    "idle": {
      "animRef": "character_idle",
      "loop": true,
      "transitionTime": 0.3
    },
    "greet": {
      "animRef": "character_greet",
      "loop": false,
      "nextAnim": "idle",           // 动画结束后的默认状态
      "transitionTime": 0.2
    },
    "interact_01": {
      "animRef": "character_interact_01",
      "loop": false,
      "nextAnim": "idle",
      "interruptible": true,         // 可被新交互打断
      "transitionTime": 0.15
    }
  }
}
```

### 3.3 状态切换规则

- `loop: true` → 动画循环播放，不会自动切换
- `loop: false` → 动画播放完毕后根据 `nextAnim` 切换到目标状态
- `interruptible: true` → 可被新交互触发打断，当前动画淡出
- `interruptible: false` → 播放期间锁定，不接受新交互

### 3.4 跨角色状态同步

若场景中存在多个角色，各自的状态机独立运行。需要同步的场景（如对话）通过 `InteractionResolver` 触发多方状态切换。

## 四、交互事件绑定

### 4.1 事件系统架构

```
交互配置文件（manifest interactions 章节）
         │
         ▼
┌─────────────────────────┐
│  InteractionRegistry     │  ← 场景加载时解析配置
│  - 注册所有交互绑定      │
│  - 管理条件 + 动作映射   │
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│  EventBus                │
│  - on("click", handler)  │
│  - on("hover", handler)  │
│  - on("drag", handler)   │
│  - on("proximity", ...)  │
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│  ConditionEvaluator      │  ← 检查触发条件
│  questCompleted?          │
│  propPlaced?              │
│  hasItem?                 │
└─────────┬───────────────┘
          │ （条件满足）
          ▼
┌─────────────────────────┐
│  ActionDispatcher        │  ← 执行动作序列
│  按顺序执行 actions[]     │
└─────────────────────────┘
```

### 4.2 交互触发流程

```
用户点击场景中的 mesh
    ↓
RayPicker 拾取 → 获取 interactionId
    ↓
InteractionRegistry 查找对应的交互配置
    ↓
ConditionEvaluator 评估条件
    ├─ 不满足 → 不触发（可选：显示提示，如"需要门禁卡"）
    └─ 满足 → ActionDispatcher 按序执行动作列表
```

### 4.3 动作执行规则

- 动作按 `actions[]` 数组顺序依次执行
- 任一动作执行失败 → 剩余动作跳过 → 记录日志
- 动作执行期间的异常 → EventBus 捕获 → 不影响后续交互

## 五、道具位置约束

### 5.1 可拖拽道具

```jsonc
{
  "props": [{
    "id": "hologram_projector",
    "pickable": true,
    "draggable": true,
    "dragPlane": "horizontal",      // "horizontal" | "vertical" | "surface"
    "dragConstraints": {
      "minPosition": [-2.0, -0.5, -3.0],
      "maxPosition": [2.0, 1.0, -1.0]
    },
    "snapTargets": [
      { "id": "desk_slot_01", "position": [1.5, 0.8, -2.0], "radius": 0.2 }
    ],
    "onPlace": "prop.projector_place"
  }]
}
```

### 5.2 拖拽流程

```
1. 鼠标按下 + 命中可拖拽道具
2. 道具跟随鼠标移动（约束在 dragPlane 上）
3. 实时检测 snapTargets 距离
   ├─ 进入吸附半径 → 道具自动吸附到目标位置
   └─ 未进入 → 自由位置
4. 鼠标松开 → 确定最终位置 → 触发 onPlace 事件
5. 保存位置到用户存档
```

### 5.3 碰撞检测

可拖拽道具之间做简单 AABB 碰撞，防止互相穿透。碰撞时道具退回上一个有效位置。

## 六、Canvas 事件穿透规则

### 6.1 穿透决策

```
鼠标事件
    │
    ├─ 当前有 React UI 面板打开？
    │    → 是 → 事件由 React UI 层处理，Canvas 不响应
    │
    ├─ 鼠标位置在 React UI 组件上方？
    │    → 是 → 事件由 React UI 层处理
    │
    ├─ 鼠标位置在 3D 可交互物体上方？
    │    → 是 → 事件由 3D 交互系统处理
    │
    └─ 鼠标位置在非交互区域（场景背景）？
         → 3D 相机操作（旋转/缩放），不穿透到下层
```

### 6.2 实现方式

- Canvas 元素设置 `pointer-events: auto`
- React UI 层通过 z-index 在 Canvas 之上
- 3D 交互系统根据拾取结果决定是否消费事件
- 非交互区域的鼠标事件被 3D 相机控制器消费，不向下传播

### 6.3 交互区域声明

主题 Manifest 中声明哪些区域接受交互：

```jsonc
{
  "interactionZones": [
    {
      "id": "desk_area",
      "boundingBox": { "min": [-1.5, 0, -3], "max": [1.5, 1.5, -1] },
      "interactionTypes": ["click", "drag"]
    }
  ]
}
```

## 七、交互异常屏蔽

### 7.1 异常场景

| 场景 | 处理 |
|------|------|
| 交互配置解析失败 | 该交互跳过，其他交互正常 |
| 拾取命中对象找不到配置 | 无响应（不报错） |
| 动画文件缺失 | 跳过动画，执行后续动作 |
| 条件评估报错 | 视为条件不满足 |
| 动作执行中途异常 | 跳过该动作，执行下一个 |

### 7.2 与主程序交互隔离

- 3D 交互系统不拦截全局快捷键
- 3D 交互系统不阻止原生右键菜单（除非在 Canvas 区域内并消费了事件）
- 3D 交互异常不导致鼠标指针卡住或主界面无响应
