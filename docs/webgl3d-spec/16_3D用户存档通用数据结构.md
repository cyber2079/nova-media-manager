## 一、数据表设计

### 1.1 表结构

```sql
-- 新增独立数据表（不修改现有表）
CREATE TABLE IF NOT EXISTS webgl3d_user_data (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    theme_id    TEXT NOT NULL,          -- 主题 ID（如 "cyber-girl-apartment"）
    slot        INTEGER NOT NULL DEFAULT 0,  -- 存档槽位（0-9）
    data        TEXT NOT NULL,          -- 存档 JSON 字符串
    created_at  TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at  TEXT NOT NULL DEFAULT (datetime('now')),
    UNIQUE(theme_id, slot)              -- 每个主题+槽位唯一
);

CREATE TABLE IF NOT EXISTS webgl3d_settings (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    key         TEXT NOT NULL UNIQUE,   -- 设置键
    value       TEXT NOT NULL,          -- 设置值（JSON）
    updated_at  TEXT NOT NULL DEFAULT (datetime('now'))
);
```

### 1.2 与原数据表的关系

```
现有数据表（不改动）            新增数据表（独立）
─────────────                  ────────
games                           webgl3d_user_data
settings                        webgl3d_settings
...                             3D 模块只读写自己的表
```

- 3D 模块的数据库迁移独立执行，失败不影响现有表
- 复用现有的 SQLite 连接和读写工具模块

## 二、存档数据结构

### 2.1 存档根 Schema

```jsonc
{
  "version": 1,                        // 存档格式版本（用于迁移）
  "themeId": "cyber-girl-apartment",
  "createdAt": "2026-07-22T14:32:01Z",
  "updatedAt": "2026-07-22T15:00:00Z",
  "playTimeSeconds": 3600,             // 累计游玩时间

  "scene": { /* 场景状态 */ },
  "camera": { /* 相机状态 */ },
  "characters": { /* 角色状态（可选） */ },
  "props": { /* 道具状态（可选） */ },
  "quests": { /* 任务状态（可选） */ },
  "inventory": { /* 背包（可选） */ }
}
```

### 2.2 scene — 场景状态

```jsonc
{
  "scene": {
    "currentSceneId": "main_room",     // 当前所在场景
    "visitedScenes": [                 // 已访问场景列表
      "main_room",
      "corridor"
    ],
    "unlockedScenes": [                // 已解锁场景
      "main_room",
      "corridor",
      "lab"
    ]
  }
}
```

### 2.3 camera — 相机状态

```jsonc
{
  "camera": {
    "position": [0.5, 1.6, 4.2],
    "target": [0.0, 1.0, 0.0],
    "fov": 60,
    "lastUpdatedAt": "2026-07-22T15:00:00Z"
  }
}
```

### 2.4 characters — 角色状态（可选）

```jsonc
{
  "characters": {
    "cyber_girl": {
      "visible": true,
      "currentAnimation": "idle",
      "position": [0.0, 0.0, -1.5],
      "affectionLevel": 5,             // 好感度（如主题设计需要）
      "interactionCount": 12,          // 交互次数
      "lastInteractionAt": "2026-07-22T14:55:00Z"
    }
  }
}
```

- 无角色主题：`characters` 字段为 `{}`
- 角色状态结构以 `characterId` 为 key

### 2.5 props — 道具状态（可选）

```jsonc
{
  "props": {
    "hologram_projector": {
      "placed": true,
      "position": [1.5, 0.8, -2.0],
      "snappedTo": "desk_slot_01",
      "lastMovedAt": "2026-07-22T14:40:00Z"
    },
    "neon_sign": {
      "placed": false,
      "position": [0.0, 0.0, 0.0],     // 默认位置
      "inInventory": true               // 在背包中
    }
  }
}
```

- 无道具主题：`props` 字段为 `{}`

### 2.6 quests — 任务状态（可选）

```jsonc
{
  "quests": {
    "find_keycard": {
      "status": "completed",           // "locked" | "unlocked" | "in_progress" | "completed"
      "currentStage": "stage_2",
      "completedStages": ["stage_1", "stage_2"],
      "unlockedAt": "2026-07-22T14:30:00Z",
      "completedAt": "2026-07-22T14:50:00Z"
    },
    "enter_lab": {
      "status": "unlocked",
      "currentStage": null,
      "completedStages": []
    }
  }
}
```

- 无任务主题：`quests` 字段为 `{}`
- 状态管理器兼容"无任务"主题，跳过任务分支逻辑

### 2.7 inventory — 背包（可选）

```jsonc
{
  "inventory": {
    "items": [
      { "itemId": "keycard", "acquiredAt": "2026-07-22T14:45:00Z" },
      { "itemId": "data_chip_01", "acquiredAt": "2026-07-22T14:52:00Z" }
    ]
  }
}
```

- 无收集系统主题：`inventory` 字段为 `{}`

## 三、存档操作

### 3.1 保存时机

| 触发条件 | 说明 |
|---------|------|
| 用户手动保存 | 显式"保存"按钮 |
| 主题切换 | 自动保存当前主题 |
| 模块关闭 | 自动保存 |
| 关键操作后 | 任务完成、道具放置、场景解锁等事件后自动保存 |
| 定时自动保存 | 每 5 分钟（可配置） |

### 3.2 数据隔离

```
用户 A
├─ 主题 1（槽 0）：{ ... 存档数据 ... }
├─ 主题 1（槽 1）：{ ... 另一个存档 ... }
├─ 主题 2（槽 0）：{ ... 完全不同主题的存档 ... }
└─ ...

不同主题的存档字段可以完全不同
（如主题 1 有 quests、主题 2 没有）
存档 Schema 由主题 manifest 定义，数据库不强制结构
```

### 3.3 存档迁移

主题升级可能导致存档格式变化：

```
读取存档 → 检查 version 字段
    ├─ version = 当前主题定义 → 直接加载
    ├─ version < 当前主题定义 → 运行迁移函数（主题 manifest 声明）
    └─ version > 当前主题定义 → 警告 + 尝试加载（不兼容字段忽略）
```

## 四、设置存储

### 4.1 webgl3d_settings 表

```
key                     | value (JSON)
────────────────────────|──────────────────────────────
webgl3d_enabled         | true / false
active_theme_id         | "cyber-girl-apartment" / null
quality_level           | "high" / "medium" / "low"
fps_mode                | "60" / "30" / "adaptive"
cache_limit_mb          | 2048
last_used_theme_ids     | ["cyber-girl-apartment", "ink-wash"]
```

### 4.2 与主设置的关系

- `webgl3d_settings` 是独立表，不影响 `settings` 表
- 3D 模块开关状态同步到主应用的设置面板显示
- 如果主应用重置设置，3D 设置独立保留（数据不丢失）
