#### 失败用例

| 编号 | 用例 | 缺陷描述 | 严重度 |
|------|------|----------|--------|
| MOV-014 | 大文件导入 (>4GB) | 4GB+ .mkv 导入时 ffmpeg 内存峰值超过 1.2GB，低配机器 (8GB RAM) 出现 OOM 风险。封面截取耗时 >60s | P1 |

2. **BUG-MOV-002**: 内嵌播放器在大分辨率 (4K) 视频时，初次加载有 2-3s 黑屏。**建议**: 添加加载指示器。

#### 未覆盖用例

- MOV-019 (视频转 WebP): 需要特定编码环境，未测试
- MOV-028 (播放器键盘控制): 等待键盘系统全局配置后测试

#### 失败/阻塞用例

| 编号 | 用例 | 缺陷描述 | 严重度 |
|------|------|----------|--------|
| MUS-011 | 歌词显示 | LRC 歌词解析不支持 `[offset:xxx]` 标签和扩展格式 (如 `[mm:ss.xx]`)。部分中日文 LRC 编码错误 (非 UTF-8 文件显示乱码) | P1 |
| MUS-022 | 无音频设备播放 | **阻塞** — 尝试播放时应用闪退。`cpal` 主机未正确处理无设备状态。panic in Rust audio thread | P0 |
| MUS-024 | 音频文件损坏 | 损坏的 .mp3 播放时，audioMotion-analyzer 出现无限循环错误日志，CPU 100% 持续 30s 后才抛出错误 | P1 |

#### 已确认问题

1. **BUG-MUS-001 (P0)**: 无音频设备时播放音乐导致应用崩溃。需在 Rust 端添加设备检测和优雅降级。
2. **BUG-MUS-002 (P1)**: 音乐可视化器在歌曲切换时有约 150ms 的频谱残留 (旧歌频谱闪烁在切换瞬间)。
3. **BUG-MUS-003 (P2)**: MiniPlayer 进度条在 duration=0 时 (如流式音频) 显示 NaN%。
4. **BUG-MUS-004 (P2)**: 播放列表超过 100 首时，shuffle 模式性能下降 (O(n) 随机算法)。

## 八、结论

Nova Media Manager v0.1.0 在核心功能上表现良好：媒体库管理、主题系统、许可证系统均基本可用。应用的整体架构（Tauri + React + Zustand + SQLite）设计合理，安全措施在桌面应用同类产品中属于中上水平。

**存在 2 个 P0 阻塞缺陷（音乐无设备崩溃、电影导入偶发状态卡死），建议修复后再发布 Beta 版本。** 其余 P1 缺陷可在 Beta 阶段迭代修复。

主题系统是应用的差异化亮点，建议在正式发布前完成 .nvtp 签名机制，为 premium 主题的商业化提供安全基础。
