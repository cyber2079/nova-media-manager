## 一、运营总体战略

### 1.1 运营哲学

```
「产品即内容，用户即社区」

Nova 的核心竞争力不只是功能，而是「沉浸式角色秘书体验」。
运营的核心任务是将这种体验转化为可传播的内容，
让每个用户都成为产品的传播节点。
```

```
验证期 (M1-M3)          启动期 (M4-M6)           增长期 (M7-M12)

   └─ 定价验证             └─ 渠道测试              └─ 规模化增长

## 五、数据运营

### 5.1 数据埋点设计

当前 `analytics.ts` 已有基础埋点。建议扩展：

| 事件 | 触发时机 | 价值 |
|------|------|------|
| `app_launch` | 应用启动 | DAU/MAU |
| `page_view` | 页面切换 | 各功能使用率 |
| `theme_switch` | 主题切换 | 主题偏好 |
| `activation_start` | 打开激活页 | 转化漏斗 |
| `activation_success` | 激活成功 | 付费转化 |
| `theme_install` | 安装主题 | 主题留存 |
| `feature_use` | 使用具体功能 | 功能热度 |
| `error` | 错误发生 | 质量监控 |
| `app_uninstall` | (无法直接埋) | 通过长时间未启动推断 |

### 5.2 关键漏斗分析

```
安装 (100%)
  │
  ├→ 首次启动 (95%)          ← 5% 安装未启动流失
  │
  ├→ 导入首批媒体 (60%)      ← 35% 空白状态流失
  │
  ├→ 探索主题系统 (40%)      ← 20% 未发现主题功能
  │
  ├→ 打开激活页 (15%)        ← 25% 不想付费
  │
  └→ 付费成功 (8%)           ← 7% 支付/价格障碍
```

**优化重点**:
1. **首次启动 → 导入媒体**: Onboarding 中直接引导导入演示文件
2. **探索主题**: 首次启动时自动展示主题切换效果
3. **激活 → 付费**: 7 天试用 + 价格优惠弹窗

### 5.3 用户留存策略

| 时间 | 留存目标 | 策略 |
|------|------|------|
| Day 1 | 80% | 确保首次体验流畅，Onboarding 引导完善 |
| Day 7 | 50% | 每日不同的打字机文案，保持新鲜感 |
| Day 30 | 30% | 自动更新内容（壁纸/文案/角色语音） |
| Day 90 | 20% | 新主题发布通知 + 社区活动 |

### 5.4 A/B 测试计划

| 测试 | 变量 | 指标 | 优先级 |
|------|------|------|------|
| 定价页设计 | A: 月付突出 B: 永久突出 | 付费率 | P1 |
| 试用期长度 | A: 7天 B: 14天 | 转化率 | P2 |
| 激活弹窗时机 | A: 启动时 B: 首次切换主题时 | 打开率 | P1 |
| 主题展示 | A: 静态图 B: GIF/视频 | 安装率 | P2 |

```

major: 重大架构变更
minor: 新功能
patch: Bug 修复
channel: alpha / beta / rc / stable

### 7.2 发布周期

| 阶段 | 周期 | 内容 |
|------|------|------|
| Alpha | 每 2 周 | 内部测试，功能迭代 |
