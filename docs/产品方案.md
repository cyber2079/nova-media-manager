# 本地HiFi音乐播放器 + 全平台游戏管理一体化软件 — 完整产品方案

## 一、产品核心定位

基于 Tauri + React + Zustand + Rust 后端的 Windows 本地媒体工具，两大核心支柱：

1. **专业本地HiFi音频播放器**：主打无损音乐精细频谱、完整标签/歌单/批量管理、歌词系统、低失真HiFi输出；
2. **全平台本地游戏扫描管理模块**：离线识别 Steam/Epic/Ubisoft/GOG/Xbox/WeGame，支持游戏一键启动与批量管理。

**差异化壁垒**：市面无同类产品同时覆盖「高颜值现代化UI、精细可调音频频谱、HiFi无损解码、全平台游戏本地识别」。

## 二、技术整体栈

### 前端（已有代码复用）
- React + Zustand（音频/曲库/歌单状态）
- Tailwind CSS 样式系统
- Web Audio API `AudioWorklet`（共享模式播放 + 频谱渲染）

### Tauri Rust 后端核心依赖
- **音频层**：`symphonia`（全格式无损解码）、`cpal` + `wasapi` crate（共享/独占双模式输出）
- **游戏识别层**：`steamlocate`（Steam）、`game-detector`（Epic/Xbox）、`winreg`（注册表）、`dirs`（系统路径）、`serde_json`（WeGame 清单解析）
- **数据存储**：SQLite（统一存放全部用户数据于 Tauri `app_data_dir`，清浏览器缓存不丢、卸载软件不丢）
- **文件处理**：tauri-plugin-fs、walkdir、id3-wasm（音频标签读写）
- **网络**：reqwest（LRCLIB 歌词、SteamGridDB 游戏封面）

## 三、模块一：HiFi音频播放系统（双模式架构）

### 3.1 统一音频管线（共享模式 & 独占模式共用）

音频数据链路：

```
音频文件 → Rust symphonia 解码 → PCM 原始数据
                                      │
                    ┌─────────────────┼─────────────────┐
                    ↓                                    ↓
              频谱渲染通道                            音频输出通道
         (PCM → 前端着色渲染)                   (PCM → 输出 Sink)
                    │                                    │
                    ↓                           ┌────────┴────────┐
            64段频谱可视化                        │                  │
        (两种模式完全一致)                  共享模式             独占模式
                                        AudioWorklet           cpal WASAPI
                                        → 系统混音器            → 声卡独占
                                        → 多软件共存            → 单音源直达
```

**设计原则：**
- 解码和频谱共用同一套 Rust 管线，两种模式零差异
- 切换模式只改输出 Sink，不重建解码链路
- PCM 数据在送入系统混音器**之前**已分一路给频谱渲染——保障共享模式下频谱不受系统重采样影响

### 3.2 模式一：WASAPI 共享模式（默认）

- **适用场景**：边听歌边打游戏、多软件同时发声
- **播放链路**：Rust symphonia 解码 → PCM → 前端 AudioWorklet → 系统混音器 → 声卡
- **优点**：多音源共存，音乐+游戏+聊天同时出声
- **缺陷**：Windows 会统一重采样音频、叠加系统音效，轻微损失高频细节

**共享模式音质补偿优化：**
1. 播放器内部使用文件原生采样率初始化解码管线，内部不做不必要的重采样
2. 若文件采样率与系统不匹配，使用高质量 SRC（sinc 插值），非系统默认线性重采样
3. 首次启动引导页提示用户手动关闭 Windows 响度均衡/空间音频（截图引导，不做自动检测）
4. 频谱计算前置——在音频送入系统混音前抓取原始 PCM 数据，保证可视化柱子精准

### 3.3 模式二：WASAPI 独占直通（HiFi 无损模式）

- **适用场景**：纯听歌、搭配 DAC 耳机、无损音乐鉴赏、频谱精细调试
- **播放链路**：Rust symphonia 解码 → PCM → cpal WASAPI 独占 → 声卡
- **限制**：开启后其他软件静音（声卡被单独锁定，系统混音器失效）
- **优势**：完全绕过系统混音、无强制重采样、原始音频直达声卡、DSD DoP 直通

**独占模式用户体验设计：**
- 切换时弹窗二次确认，写明副作用
- 播放器底部常驻小字提示「当前独占音频，其他应用静音」
- 一键切回共享模式，不中断播放

### 3.4 频谱可视化（双模式效果一致）

- 基础：16 段频谱
- 高阶：64 段高精度频谱，自定义增益/平滑/柱子尺寸/色彩 UI 面板

**核心技术保障**：频谱数据在 Rust 解码后、送入音频输出 Sink **之前**抓取。不管共享还是独占，读取的都是未经系统修改的原始 PCM 波形。两种模式下 64 段频谱显示效果几乎无差别，仅最终外放音质有细微差距。

### 3.5 进阶音频功能

**多设备音频分流：**
- 软件内枚举全部音频输出设备
- 播放器输出到外置 DAC（WASAPI 独占），游戏/系统声音走主板声卡（共享）
- 实现：音乐无损独占 + 游戏声另一路出声，互不干扰

**31 段数字 PEQ 均衡器：**
- 内置电子/人声/古典/Phonk 等预设曲线
- 零相位升频插值、全局音量标准化、动态响度压缩

**DSD 解码：**
- DSD256/DSD512 硬软解切换（symphonia 支持有限，需单独评估）
- MQA 基础解析

**ASIO 低延迟通道（后期迭代）：**
- ASIO 驱动支持多应用同时接入音频通道
- 仅支持专业声卡，普通板载声卡无驱动

## 四、模块二：全平台本地游戏离线识别系统

### 4.1 支持识别平台与底层解析方案（全部离线）

| 平台 | 实现方案 | 识别数据源 | 更新兼容性 |
|------|--------|------------|-----------|
| Steam | steamlocate Rust 库 | steamapps/*.acf VDF 清单 | 游戏更新、多磁盘库完全兼容 |
| Epic / Xbox PC Game Pass | game-detector 库 | 注册表 + Manifest JSON | 客户端自动更新无干扰 |
| Ubisoft Connect (Uplay) | 自研解析 | 注册表游戏 GUID + ProgramData 全局清单 | 迁移游戏自动同步路径 |
| GOG Galaxy | 自研解析 | Windows 注册表游戏键值 | 无加密，读取稳定 |
| WeGame（腾讯） | 自研 JSON 解析 | %LocalAppData%/WeGame/games.json + 注册表兜底 | 游戏自动更新、迁移不影响识别 |

### 4.2 统一扫描流程

1. 分优先级串行扫描各平台，成熟 crate 优先执行，自研解析后置
2. 读取所有平台游戏元数据，统一标准化为通用 Game 结构体（平台标识、唯一ID、游戏名、安装目录、启动exe、本地封面路径）
3. 合并去重，存入本地 SQLite 缓存，二次打开直接读取缓存，无需重复扫描
4. 增加文件存在校验，自动过滤已卸载、路径失效游戏

### 4.3 游戏素材补齐（联网按需拉取，离线可用）

1. 优先读取客户端本地自带封面缓存
2. 联网调用 SteamGridDB 公共 API，通过各平台唯一 ID 拉取高清封面、背景图、图标
3. 图片本地持久缓存，重复使用不重复请求网络

## 六、软件运行逻辑总流程

1. **软件启动**
   - 初始化全局单例音频上下文（Rust 端 symphonia 解码器 + cpal 输出设备枚举）
   - 读取本地存储（SQLite）和用户偏好配置

2. **曲库加载**
   - 读取本地音乐库元数据，分页渲染列表/卡片（大卡片 20 条/页，小卡片 30 条/页）

3. **游戏扫描（按需触发，不强制开机扫描）**
   - 用户点击「游戏库」页面时，执行多平台串行扫描，写入缓存数据库

4. **播放歌曲流程**
   - Rust 端 symhonia 解码 → PCM 数据
   - 频谱通道：PCM → 前端着色渲染（16/64 段）
   - 音频通道：PCM → AudioWorklet（共享）/ cpal WASAPI（独占）→ 声卡
   - 歌词：读取本地缓存 → 无缓存异步请求 LRCLIB → 存入 SQLite

5. **关闭软件**
   - 持久化所有数据库缓存、释放音频资源

## 七、兼容性与风险控制

1. **游戏客户端更新兼容**：所有游戏识别均读取客户端本地配置文件，游戏补丁/更新不修改清单/注册表路径，识别逻辑完全不受影响
2. **权限需求**：仅普通用户权限，无需管理员，Tauri 默认文件/注册表读取权限足够
3. **性能优化**：频谱 50ms 刷新 20fps、大图封面本地缓存
4. **断网降级**：无网络时音频播放、本地曲库、本地游戏列表全部正常使用，仅关闭在线歌词、游戏封面下载
5. **安装包体积**：纯 Rust 解码 + 游戏解析库，无大型二进制，整体安装包控制在 35-50MB

## 九、核心竞争优势总结

1. **音频层面**：兼顾普通听歌与 HiFi 发烧友，精细可调频谱解决电子/Phonk 用户痛点，双模式架构让"边听歌边打游戏"和"纯 HiFi 鉴赏"两不误
2. **游戏管理层面**：唯一同时覆盖 Steam/Epic/Uplay/GOG/Xbox/WeGame 全平台本地离线识别的音乐播放器
3. **体验层面**：现代化 React UI，对比 Foobar2000/AIMP 老旧原生界面，视觉与交互优势巨大
4. **轻量化**：Tauri 纯本地运行，无后台上传文件，隐私性优于流媒体软件
5. **频谱精准度**：前置抓取 PCM 方案确保共享模式下频谱精度与独占模式一致，电子/Phonk 用户视觉体验不打折
