## 一、测试策略

### 1.1 测试层次

```
┌──────────────┐
│   E2E 测试    │  ← 用户真实操作流程 (30 条)
├──────────────┤
│  集成测试     │  ← 前后端交互 + 数据库 (45 条)
├──────────────┤
│  单元测试     │  ← 独立函数/组件 (80 条)
├──────────────┤
│  安全渗透     │  ← 注入/篡改/逆向 (14 条)
├──────────────┤
│  性能基准     │  ← 启动/内存/帧率 (10 条)
└──────────────┘
```

### 1.2 优先级定义

| 级别 | 定义 | 数量 |
|------|------|------|
| **P0** | 核心功能，阻塞发布 | 42 |
| **P1** | 重要功能，影响体验 | 89 |
| **P2** | 边缘场景，可延后 | 58 |
| **P3** | 优化项，锦上添花 | 28 |

### 1.3 用例格式

每条用例包含：ID、模块、优先级、前置条件、操作步骤、预期结果、实际结果、状态。

#### TC-MOV-001 | 导入单个电影文件
- **优先级**: P0
- **前置条件**: 本地存在 ≥1 个视频文件 (.mp4/.mkv/.avi)
- **步骤**:
  1. 点击导航栏「电影」
  2. 点击导入按钮（或拖拽文件到窗口）
  3. 选择 1 个视频文件
  4. 等待处理完成
- **预期结果**:
  - 文件出现在电影列表中
  - 自动生成缩略图封面
  - 正确读取分辨率、时长、文件大小
  - 状态从 "processing" 变为 "ready"
  - Toast 提示导入成功
- **验证点**:
  - [ ] 封面缩略图生成正确
  - [ ] 时长格式为 mm:ss 或 hh:mm:ss
  - [ ] 文件大小显示正确单位 (MB/GB)
  - [ ] SQLite movies 表记录正确
  - [ ] 不重复导入相同路径文件
